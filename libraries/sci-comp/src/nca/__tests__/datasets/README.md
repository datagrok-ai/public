# NCA Reference Datasets

Input CSV datasets used to generate PKNCA reference values
(`../fixtures/*.json`) and to drive the validation harness in
`__tests__/reference-suite.test.ts`.

CSV files are committed as **source artifacts**: they are produced once and
not regenerated by the fixtures script. The R script in
`scripts/generate-nca-fixtures.R` reads these CSVs, runs PKNCA, and writes the
JSON fixtures.

## 01_theoph.csv — Theophylline pharmacokinetics

Classical PK dataset shipped with R. 12 subjects given a single oral dose of
theophylline; concentrations sampled over ~25 hours.

**Columns:** `Subject, Wt, Dose, Time, conc`

**Units:**
- `Wt` — body weight in kg
- `Dose` — oral dose in mg/kg (already normalised per subject)
- `Time` — hours post-dose
- `conc` — theophylline concentration in mg/L

**Route:** extravascular (oral) — apparent clearance and volume parameters
will be `CL/F` and `Vz/F`.

**Reference:** Boeckmann, A. J., Sheiner, L. B. and Beal, S. L. (1994),
*NONMEM Users Guide: Part V*. NONMEM Project Group, University of California,
San Francisco. Provided with R via the `datasets` package.

## 02_indometh.csv — Indomethacin IV pharmacokinetics

Classical PK dataset shipped with R. 6 subjects given an intravenous bolus
injection of 25 mg indomethacin; concentrations sampled over 8 hours.

**Columns:** `Subject, time, conc`

**Units:**
- `time` — hours post-dose
- `conc` — indomethacin concentration in μg/mL (= mg/L)

**Dose:** 25 mg IV bolus, identical for all subjects (not stored in CSV — must
be supplied by the consumer).

**Route:** IV bolus — clearance and volume parameters are `CL` and `Vz` (no
bioavailability factor).

**Reference:** Kwan, K. C., Breault, G. O., Umbenhauer, E. R., McMahon, F. G.
and Duggan, D. E. (1976), *Kinetics of indomethacin absorption, elimination,
and enterohepatic circulation in man.* Journal of Pharmacokinetics and
Biopharmaceutics, **4**, 255–280. Provided with R via the `datasets` package.

## 03_rat_simple.csv — Synthetic rat oral PK

Synthetic single-dose oral PK data for 8 rats, generated from a one-compartment
model with first-order absorption + log-normal residual error.

**Columns:** `Subject, Dose, Time, conc`

**Units:**
- `Dose` — mg per rat (single oral dose, identical across rats: 2.5 mg)
- `Time` — hours post-dose
- `conc` — concentration in mg/L

**Route:** extravascular (oral) — apparent clearance and volume parameters
will be `CL/F` and `Vz/F`.

**Generation parameters** (in `scripts/generate-dataset-03-rat-simple.R`):
- Population means: `ka = 1.5 /hr`, `ke = 0.3 /hr` (t½ ≈ 2.31 hr),
  `V = 0.375 L`, `F = 0.6`.
- Inter-subject variability (log-normal SD): `ka, ke, V` — 0.20; `F` — 0.10.
- Residual error: 10% proportional + 0.01 mg/L additive.
- Seed: 42.
- Timepoints: 0.25, 0.5, 1, 2, 4, 6, 8, 12 hours.

**Why synthetic for Phase 0:** the planned full multiroute / multidose /
multi-occasion rat dataset is deferred to Phase 2 (datasets 04 and 06). This
simplified dataset provides an oral-route, multi-subject test case for the
Phase 1 validation gate without blocking on access to real data. See Risk
Register R7 in `docs/nca_development_plan_v2.md`.

**Regeneration:** the script enforces guards (no zero concentrations,
monotonic tails for the last 3 points). If guards trip after parameter
changes, either tighten the noise model or change the seed.
