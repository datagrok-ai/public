## Code Style Rules

- 2-space indentation
- Single quotes for strings
- Semicolons required
- Windows line endings (CRLF)
- TypeScript strict mode
- File naming: kebab-case (`my-component.ts`)
- Package folder naming: PascalCase (`PowerGrid`, `BiostructureViewer`)
- Package name in package.json: kebab-case with scope (`@datagrok/power-grid`)
- Semantic versioning: `X.Y.Z` or `X.Y.Z-rc` or `X.Y.Z-rc.N`
- Never edit files ending in `.g.ts` or `.api.g.ts` — these are auto-generated by `grok api`
