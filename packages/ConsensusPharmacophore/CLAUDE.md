# ConsensusPharmacophore — guidance for coding agents

## What this package is

A **single-button Datagrok app** that runs a 5-stage ligand-based 3D consensus pharmacophore pipeline on a
list of PDB IDs. Mid-pipeline UI gate (cluster picker) lets the user reject conformationally mixed inputs.

Reference blueprint: [`blueprint/ligand-based-3d-pharmacophore-datagrok/blueprint.md`](../../blueprint/ligand-based-3d-pharmacophore-datagrok/blueprint.md).

## Pipeline shape

```
TS Stage 1  → enrich PDB metadata, cofactor-filter ligands (browser, uses RCSB Data API + WASM-RDKit)
Py Stage 2a → Kabsch pass 1, global Cα (MDAnalysis)
TS picker   → ui.dialog with conformational clusters
Py Stage 3  → pocket isolation (5 Å cutoff default; DBSCAN advanced)
Py Stage 2b → Kabsch pass 2, pocket Cα only — overwrites 2a transforms
Py Stage 4  → ProLIF protein-ligand interactions (pdbfixer + MDA + ProLIF; one row per detected interaction)
              [SMARTS-only version archived at files/_reference/04_extract_features_smarts_only.py]
Py Stage 5a → per-family k-means consensus (n_init=10, random_state=42)
TS Stage 5b → fake-PDB-HETATM block (B-factor = frequency) — Mol* CPK-colors by element
```

Orchestrator: `src/orchestrator.ts`. Each `grok.functions.call` target is a constant in `src/function-names.ts`
to keep string drift from `#name:` directives silent-fail-proof.

## Hard rules

1. **Never edit `package.g.ts` or `package-api.ts`** — regenerated by `grok api`.
2. **Inter-stage interchange is `(pdb_id, original_pdb, transform_4x4_json)`** — original PDB stays
   canonical; consensus-frame coords are reconstructable, never persisted twice (research F7d).
3. **WASM RDKit** for browser-side SMILES validation and 2D thumbnails — via
   `getRdKitModule()` from `@datagrok-libraries/bio/src/chem/rdkit-module`. Do NOT depend on `packages/Chem`
   directly.
4. **Cross-package viewers** — instantiate Mol\* with `await df.plot.fromType('Biostructure', {pdb: pdbStr})`.
   No source import of `@datagrok/biostructure-viewer` runtime symbols; the package is a *devDep* (types-only).
5. **External HTTP** — only `grok.dapi.fetchProxy(...)`. Never raw `fetch()` (CORS).
6. **Python scripts** — every script reuses BSV's `#environment:` line verbatim (Datagrok caches the conda env
   by MD5 of the env string; exact match shares the env with BSV). Append `scikit-learn>=1.3` for Stage 5a.
7. **Single `DG.TableView`** for the whole pipeline. Replace `view.dataFrame = newDf` between stages; the
   docked Mol\* viewer survives the swap (verified in Phase 0 V2). Defensive `viewer.setOptions({pdb: ...})`
   re-assertion after each swap.

## Pharmacophore family taxonomy (single source of truth)

Maps 1:1 to `packages/Chem/files/pharmacophore-features.csv` family codes. The CSV is bundled as a copy at
`files/pharmacophore-features.csv` — refresh via `scripts/_sync_smarts.sh`.

| Datagrok code | Rendered element (CPK) | resName | 2D widget hex |
|---|---|---|---|
| D | N (blue)   | HBD | #2196F3 |
| A | O (red)    | HBA | #E53935 |
| a | S (yellow) | ARO | #4CAF50 |
| H | C (grey)   | HYD | #FFEB3B |
| P | NA (violet)| POS | #00BCD4 |
| N | CL (green) | NEG | #FF9800 |

## Test categories

`grok test --category Pharmacophore` runs everything. Per-stage unit tests, cross-stage contract tests, and
the golden-snapshot regression test. Tests that hit RCSB use `skipReason` when the endpoint is unreachable —
keep the pipeline green on offline CI.
