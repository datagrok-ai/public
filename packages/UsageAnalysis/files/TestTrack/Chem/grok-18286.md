# GROK-18286: Scaffold tree: closing colored scaffold causes errors and breaks Scatterplot

## Summary
Closing a colored scaffold in the Scaffold Tree viewer causes errors and breaks the Scatterplot,
when the scaffold color column was previously set as the color in the Scatterplot.

## Steps to reproduce
1. Open a table with molecules (e.g. demo smiles data).
2. Add a Scaffold Tree viewer and add/generate scaffolds.
3. Enable coloring for a scaffold (a scaffold color column is applied to the molecule column).
4. Add a Scatterplot and set its color to the scaffold color column.
5. Close (remove) the colored scaffold in the Scaffold Tree.

## Expected vs actual
- Expected: the scaffold closes cleanly; the Scatterplot handles the color column removal gracefully (color resets, no errors).
- Actual: errors are thrown (reported error: null) and the Scatterplot breaks.

## Notes
- Component: Chem (Scaffold Tree) interacting with Scatterplot color-coding.
