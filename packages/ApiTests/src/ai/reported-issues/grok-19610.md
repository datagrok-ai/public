# GROK-19610: Box plot zoom-values-by-filter toggle

Regression coverage for the box-plot "Zoom Values by Filter" property.
Closed in Feb 2026 with a tooltip-only clarification; we still want to pin
the property key, the round-trip through the look bag, and the non-empty
description that was the actual user-facing fix.

## Triage scope

The original report: with **Zoom Values by Filter** applied to the same
column used on the Y-axis, disabling the toggle did not restore the
viewport. Resolution shipped as a tooltip clarification on the property,
not a behavior change. Pixel-level / viewport-restoration assertions are
out of scope for the JS API suite.

## Cases

| # | Case                                                          | What it pins                                                |
|---|---------------------------------------------------------------|-------------------------------------------------------------|
| 1 | Build `df.plot.box({value, category})`, toggle the prop true→false | `props['zoomValuesByFilter']` and `look.zoomValuesByFilter` agree on each toggle. |
| 2 | Apply a filter, toggle the prop several times                 | Round-trip survives an active filter; no exceptions thrown. |
| 3 | `viewer.getProperties()` carries the property                 | Found by name `zoomValuesByFilter`; `description` is a non-empty string (the tooltip the ticket added). |

## API surface exercised

- `df.plot.box({value, category})`
- `IBoxPlotSettings.zoomValuesByFilter`
- `Viewer.props[...]` (read/write)
- `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]`
- `Property.name`, `Property.description`
- `DataFrame.filter.set`, `DataFrame.filter.setAll`

## Out of scope

- Verifying that disabling the toggle actually restores a prior zoom range.
  The shipped fix was tooltip text; behavior was not changed.
- Visual / pixel rendering of the box plot.
