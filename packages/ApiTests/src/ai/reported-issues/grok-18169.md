# GROK-18169: PC plot showMin and showMax

Regression coverage for the PC plot min/max and label display flags. The
combined `showMinMax` flag previously worked, but the underlying individual
`showMin` / `showMax` toggles did not — the fix wires both. Verified
properties on `IPcPlotSettings`: `showMinMax` (`d4.ts:2778`),
`showLabels` (`d4.ts:2780`), `showDensity` (`d4.ts:2776`).

## Triage scope

The user-facing change ensures the JS API surface for `IPcPlotSettings` flags
round-trips through `props` and `getOptions(true).look`. Visible drawing
differences (whether/how labels and min/max marks are actually painted) are
out of scope — assert the state-machine only.

## Cases

| # | Case                                                                                            | What it pins                                                                                       |
|---|-------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------|
| 1 | `df.plot.fromType(VIEWER.PC_PLOT)`, toggle `showMinMax` true/false twice                        | `props['showMinMax']` and `look.showMinMax` agree on each toggle.                                  |
| 2 | `viewer.getProperties()` exposes `showMinMax`                                                   | Found by name; description checked best-effort (non-empty preferred, but tolerated if missing).    |
| 3 | Schema tripwire: probe for `showMin` / `showMax` as separate descriptors and round-trip if so   | If present, each round-trips through `props` and `look`. If absent, document and pass.             |
| 4 | Toggle `showLabels` and `showDensity` independently                                             | Each round-trips through `props` and `look` without affecting the other.                           |

## API surface exercised

- `DG.Viewer.pcPlot(df)` / `df.plot.fromType(VIEWER.PC_PLOT)`
- `IPcPlotSettings.showMinMax`
- `IPcPlotSettings.showLabels`
- `IPcPlotSettings.showDensity`
- Best-effort: `showMin` / `showMax` as runtime probes
- `Viewer.props[...]` (read/write)
- `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]`
- `Property.name`, `Property.description`

## Out of scope

- Verifying that min/max ticks or labels are actually rendered on the canvas.
- Visual / pixel-level assertions on the PC plot.
- `viewer.dataFrame === df` identity check (unreliable across viewer creation paths).

## Risks

- `showMin` / `showMax` may not appear on the JS interface — kept as a
  best-effort runtime probe; absence does not fail the regression.
- `getProperties()` may not enumerate every newly-wired prop in older
  descriptors; the second case demotes to a look-bag round-trip in that case
  so the regression remains meaningful.
