# GROK-19466: Trellis Bar chart global scale + aggr

**JIRA:** GROK-19466 (Done, fix on master in 1.27.0)
**JS API source:** `public/js-api/src/viewer.ts:299` (`DG.Viewer.trellisPlot`), `public/js-api/src/interfaces/d4.ts:3570` (`ITrellisPlotSettings` — `innerViewerLook` at 3586, `globalScale` at 3605), `public/js-api/src/const.ts:696` (`DG.VIEWER.TRELLIS_PLOT`), `public/js-api/src/const.ts:673` (`DG.VIEWER.BAR_CHART`)
**Date:** 2026-04-29

## What we are testing

Regression coverage for the Trellis-plot bug where, with the inner viewer set to `Bar chart` and `globalScale: true`, swapping `valueAggrType` (avg / sum / min / max / count / med / q1 / q3 / stdev) used to leave the trellis stuck or broken. The fix landed in 1.27.0; this suite pins the JS API surface so the regression cannot return.

All cases construct a detached `DG.Viewer.trellisPlot(...)` (no `TableView` is opened), so we never depend on canvas geometry or first-paint. Assertions read state via `viewer.getOptions(true).look` — never pixels.

## Test cases

1. **construct trellis Bar chart with globalScale=true** — `DG.Viewer.trellisPlot(df, {viewerType: 'Bar chart', xColumnNames: ['race'], yColumnNames: ['sex'], globalScale: true, innerViewerLook: {...}})` does not throw, `viewer.type === DG.VIEWER.TRELLIS_PLOT`, and the JSON envelope round-trips `viewerType`, `globalScale`, and `innerViewerLook.{valueAggrType,valueColumnName}`.
2. **switch valueAggrType to avg under globalScale** — after construction, `viewer.setOptions({innerViewerLook: {valueAggrType: 'avg', ...}})` does not throw and `getOptions(true).look.innerViewerLook.valueAggrType === 'avg'`.
3. **cycle through every aggr type under globalScale=true** — for each of `avg / sum / min / max / count / med / q1 / q3 / stdev`, swap and assert no throw + state.
4. **toggle globalScale between true/false after each aggr change** — for each aggr, swap aggr then flip `globalScale` twice and assert the toggled state plus the aggr persist.
5. **globalScale=false then switch aggr then globalScale=true** — interleaves `globalScale` flips with aggr changes; final state is `globalScale: true` with `valueAggrType: 'max'`.

## Why state, not pixels

`innerViewerLook` is typed `any` on the JS side and is backed by a Dart `LookAndFeel` object that is rebuilt when reassigned via `setOptions`. Setting it does not necessarily trigger an immediate redraw, so we assert via the serialized JSON envelope (`getOptions(true).look`) — round-7/8/9 lessons re. canvas first-paint geometry and never-attached viewers apply here.

## Preconditions

- `grok.data.demo.demog(40)` is available — provides `race`, `sex`, `age` columns used by the trellis split and inner Bar chart.
- No server-side state is created. No `TableView` is opened. No `close()` is called on a never-attached viewer (round-5 platform issue).

## Out of scope

- Rendering / pixel sampling on the trellis grid.
- The Dart-side `BarChartLook` property grid — covered Dart-side.
- Other inner viewer types (Histogram, BoxPlot, Summary) — only Bar chart is in scope per the JIRA triage.
- `innerViewerLooks` map (per-cell overrides) — separate API, not regressed.
