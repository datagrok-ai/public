# GROK-13206: Density plot backColor

Regression coverage for `IDensityPlotSettings.backColor` round-trip on a
standalone density plot.

## Triage scope

The original bug: setting `backColor` (a `number`) on a density plot did not
take effect â€” the value was dropped between `setOptions` and the look bag.
The fix makes the value survive a `setOptions` / `getOptions(true).look`
round-trip on a standalone DensityPlot. `IDensityPlotSettings.backColor:
number` is verified at `public/js-api/src/interfaces/d4.ts:1558`.

These tests deliberately limit themselves to **state machine integrity** of
the property bag: the numeric ARGB written to `setOptions` comes back via
`getOptions(true).look`, the type stays `number`, and the descriptor surface
lists `backColor` (best-effort with a look-bag fallback for older builds).

Pixel-level paint correctness â€” i.e., that the canvas actually ends up with
the requested background color â€” is **not asserted here** and must be
validated in manual / image-diff QA.

## Cases

| # | Case                                                                    | What it pins                                          |
|---|-------------------------------------------------------------------------|-------------------------------------------------------|
| 1 | `df.plot.density(...)` then `setOptions({backColor: 0xFFCCEEFF})`        | `look.backColor` equals the ARGB and is a `number`    |
| 2 | `setOptions({backColor: DG.Color.fromHtml('#ffcceeff')})`                | `DG.Color.*` constants round-trip; type stays `number` |
| 3 | `DG.Viewer.densityPlot(df, {backColor: ...})` initial settings           | `backColor` flows through the constructor path        |
| 4 | `getProperties()` lists `backColor` (with look-bag fallback)             | Descriptor surface exposes the prop                   |

## API surface exercised

- `df.plot.density(settings)` and `DG.Viewer.densityPlot(df, settings)`
- `IDensityPlotSettings.backColor` (`d4.ts:1558`)
- `DG.Color.fromHtml`
- `viewer.setOptions(...)`, `viewer.getOptions(true).look`
- `viewer.getProperties()`

## Out of scope / known gaps

- **Trellis paths are excluded.** Some Trellis variants do not preserve
  `backColor` on `innerViewerLook`. This test is restricted to standalone
  DensityPlot.
- **No pixel-level assertion.** JS API can confirm the property bag is wired
  correctly but cannot see whether the canvas was actually painted with the
  requested background color.
- **ARGB sign coercion tolerated.** A 0xFF-prefixed ARGB int has the top bit
  set and may come back as a negative `number` when round-tripped through
  Dart. Comparison uses `actual === expected || (actual | 0) === (expected | 0)`.
- **`viewer.dataFrame === df` is unreliable** on density plot, so it is not
  asserted.
