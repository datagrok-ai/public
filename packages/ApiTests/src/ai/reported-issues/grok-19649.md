# GROK-19649: Bar chart orientation and category width

Regression coverage pinning the JS API round-trips for the bar chart label
sizing / orientation / category width properties referenced in the ticket.

## Triage scope

The fix touched bar chart label rendering, orientation handling, and the
category width sizing. The JS API surface exposes these through
`IBarChartSettings`:

- `orientation` (`d4.ts:826`) — `'horizontal'` | `'vertical'`
- `maxCategoryWidth` (`d4.ts:878`)
- `categoryValueWidth` (`d4.ts:880`)
- `showCategoryValues` (`d4.ts:835`)

Pixel-level rendering / actual label layout is out of scope for the JS API
suite; we pin only the property contract and round-trip.

## Cases

| # | Case                                                          | What it pins                                                |
|---|---------------------------------------------------------------|-------------------------------------------------------------|
| 1 | Build `df.plot.bar`, flip `orientation` between its two values | `props['orientation']` and `look.orientation` agree on each flip. |
| 2 | `setOptions({maxCategoryWidth: 200, categoryValueWidth: 80})` | Both reflected in `look` and `props`.                       |
| 3 | Toggle `showCategoryValues` true→false                        | `props['showCategoryValues']` and `look.showCategoryValues` agree. |
| 4 | `getProperties()` carries the four prop names (best-effort)   | At least one of `orientation`, `maxCategoryWidth`, `categoryValueWidth`, `showCategoryValues` is runtime-registered; descriptions are checked when present. |

## API surface exercised

- `df.plot.bar({value, split})`
- `DG.Viewer.barChart(df, {...})`
- `IBarChartSettings.{orientation, maxCategoryWidth, categoryValueWidth, showCategoryValues}`
- `Viewer.props[...]` (read/write)
- `Viewer.setOptions({...})`
- `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]`
- `Property.name`, `Property.description`

## Out of scope

- Asserting specific default values for any of the four props.
- Asserting `viewer.dataFrame === df` — unreliable across the Dart→JS
  wrapper boundary.
- Visual / pixel rendering of bar chart labels.

## Risks

- Some IBarChartSettings props are typed but not runtime-registered through
  `getProperties()`; the property-discovery case is intentionally
  best-effort.
