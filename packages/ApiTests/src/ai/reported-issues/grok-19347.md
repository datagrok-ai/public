# GROK-19347: Density plot xMin/xMax/yMin/yMax

## Summary

Density plot gained `xMin`, `xMax`, `yMin`, `yMax` axis range controls. The
properties are declared on `IDensityPlotSettings` (`d4.ts:1542-1548`).
Resolved in 1.27.0/1.28.0.

## What is verified

1. `df.plot.density({x:'age', y:'height'})` followed by
   `setOptions({xMin:0, xMax:100, yMin:0, yMax:200})`. All four keys round-trip
   through `getOptions(true).look`.
2. Inverted ranges (`xMin:100, xMax:0`) do not throw and the viewer survives.
   We deliberately do NOT pin a clamped value — the API is allowed to either
   clamp or reject.
3. `setOptions({xMin: NaN})` does not throw.
4. `getProperties()` is best-effort: when descriptors enumerate the four
   props, the test asserts they are present; otherwise it falls back to the
   round-trip check.

## Risks / non-goals

- Density plot has no `resetView()` JS method — do not call it.
- The computed axis min/max are not read from any canvas-derived source —
  only `getOptions(true).look` is treated as authoritative.
- `viewer.dataFrame === df` is unreliable on density plot, so it is not
  asserted.
