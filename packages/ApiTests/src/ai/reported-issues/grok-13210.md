# GROK-13210: Trellis Box plot bin color schema

Low-priority regression coverage for a Trellis-plot bug fixed in Jul 2025:
when the inner viewer was a Box plot and Bin Color was set, the inner cells
rendered as empty plots. The component owns the regression risk on the
`innerViewerLook` JSON envelope; the visible symptom is canvas-only.

## Triage scope

The user-facing symptom (empty canvas inside each trellis cell) is a visual
regression that JS-only ApiTests cannot detect. This file lightly pins the
**schema round-trip** for `binColor*` keys so a regression that drops or
renames them on `innerViewerLook` is caught at the property layer. Pixel
correctness must be re-checked via manual / image-diff QA.

## Cases

| # | Case                                                                         | What it pins                                                              |
|---|------------------------------------------------------------------------------|---------------------------------------------------------------------------|
| 1 | `DG.Viewer.trellisPlot(df, {viewerType: 'Box plot', innerViewerLook: {...}})` | Viewer constructs without throw; `binColorColumnName`/`binColorAggrType` round-trip via `getOptions(true).look.innerViewerLook`. |
| 2 | Toggle `binColorAggrType` through all aggregate types (`sum`, `avg`, `min`, `max`, `count`, `med`, `q1`, `q3`, `stdev`) via `viewer.setOptions({innerViewerLook: {...}})` | Each `setOptions` call does not throw, and the value is persisted through `getOptions(true).look.innerViewerLook`. |

## API surface exercised

- `DG.Viewer.trellisPlot(t, options)` (`viewer.ts:299`)
- `DG.VIEWER.TRELLIS_PLOT` (`const.ts`)
- `IBoxPlotSettings.binColor`, `IBoxPlotSettings.binColorColumnName`,
  `IBoxPlotSettings.binColorAggrType`
  (`public/js-api/src/interfaces/d4.d.ts:151-153`)
- `ITrellisPlotSettings.innerViewerLook`
  (`public/js-api/src/interfaces/d4.d.ts:1512`)
- `Viewer.setOptions({...})`, `Viewer.getOptions(true).look`

## Out of scope / unverifiable

- **Visual emptiness of the inner Box plot cells.** The original report's
  symptom is "the plot draws nothing", which is canvas-only and unreachable
  from JS. This test is a "schema sanity" tripwire only.
- Aggregation correctness for each `binColorAggrType` against numeric data;
  only that the setter accepts each literal and the value is persisted on
  `innerViewerLook`.
- Color-axis rendering, legend population, hover/highlight behavior.

## Risks

Low. This is a pure JSON-envelope round-trip on `innerViewerLook` and does
not depend on an attached `TableView`, layout, or rendering. If the actual
prop key on the inner Box plot's look class (see
`core/client/d4/lib/src/viewers/box_plot/box_plot_look.dart`) ever diverges
from `IBoxPlotSettings.binColor*`, the test should be re-pointed accordingly
— `viewer.getProperties()` is the canonical way to discover the live name.
