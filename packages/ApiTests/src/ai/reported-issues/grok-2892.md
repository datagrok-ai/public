# GROK-2892: Pie chart onClick filter semantics

Regression coverage for GROK-2892. With `onClick='Filter'`, clicking a Pie
chart slice should filter all rows in that category. The actual click-driven
filter pipeline is UI-only and not reachable from the JS API, so this file
pins the **property side**: the `onClick` enum value round-trips cleanly
through the factory, `setOptions`, the `props` bag, and the persisted `look`
snapshot, and `getProperties()` actually exposes the `onClick` property.

## Cases

| # | Case | What it pins |
|---|------|--------------|
| 1 | `DG.Viewer.pieChart(df, {category, onClick:'Filter'})` + `setOptions({onClick:'Filter'})` | `v.props.onClick === 'Filter'` and `getOptions(true).look.onClick === 'Filter'`. |
| 2 | All three `RowGroupAction` values (`'Select'`, `'Filter'`, `'None'`) via `setOptions` | Each value reflects in both `v.props.onClick` and `look.onClick`. |
| 3 | `v.getProperties()` lists `onClick`; description (when present) is a non-empty string | Public property surface for `onClick` exists; description is best-effort. |

## API surface exercised

- `DG.Viewer.pieChart(df, settings)` (`viewer.ts:279`)
- `IPieChartSettings.onClick` (`d4.ts:2883`)
- `RowGroupAction` enum string values: `'Select'`, `'Filter'`, `'None'` (`d4.ts:987-991`)
- `Viewer.setOptions({...})`, `Viewer.props`, `Viewer.getOptions(includeDefaults)`, `Viewer.getProperties()`
- `Viewer.close()` (indirectly via `tv.close()`)

## Risks / caveats

- Actual click-driven filter behavior is UI-only — we don't simulate the click,
  only pin the `onClick` property surface that the click handler reads.
- `df.plot.pie` does not exist — the test uses the `DG.Viewer.pieChart`
  factory.
- `viewer.dataFrame === df` is unreliable across the Dart→JS wrapper boundary,
  so we don't assert on it.
- `viewer.close()` on a never-attached viewer throws — every viewer in this
  file is attached via `tv.addViewer(v)` and torn down via `tv.close()`.
- The property `description` is treated as best-effort: we assert it's a
  non-empty string only when the platform actually returns one.

## Out of scope

- Pixel-level rendering, hit testing, slice selection.
- Cross-viewer filter coordination — covered by GROK-19970 (filter reset on
  close).
