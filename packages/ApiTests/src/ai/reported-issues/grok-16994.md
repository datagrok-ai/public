# GROK-16994: Scatter plot whiskers columns

Regression coverage for whisker (StdDev / errorbar) column support on
the Scatter plot, shipped in 1.23.0.

## Triage scope

The ticket adds a public API for whisker columns on the Scatter plot:

- `xWhiskerMinColumnName`, `xWhiskerMaxColumnName`, `xWhiskerRangeColumnName`
- `yWhiskerMinColumnName`, `yWhiskerMaxColumnName`, `yWhiskerRangeColumnName`

It also auto-detects whisker columns from name suffixes
(` min`, ` max`, ` range`, plus the variants ` whisker`, ` error`)
and from column tags whose key contains `min` / `max` / `range`.
Min/max takes priority over range when both are available. Visual
rendering of the whisker bars is out of scope for these tests; we pin
state machine integrity (props get set, persist through
`getOptions(true).look`, and `df.plot.scatter` does not throw).

The pixel-level rendering is covered by manual / image-diff suites
under `core/client/d4/lib/src/viewers/scatterplot/`.

## Cases

| # | Case                                                              | What it pins                                                  |
|---|-------------------------------------------------------------------|---------------------------------------------------------------|
| 1 | df with `<base> min` / `<base> max` columns on both axes          | Auto-detection populates `*WhiskerMinColumnName` / `Max...` for both axes; range stays empty. |
| 2 | df with only `<base> range` column                                | `*WhiskerRangeColumnName` set; min/max stay empty.            |
| 3 | Manual `xWhiskerMin/MaxColumnName` with non-suffix names          | Round-trip via `props` and `getOptions(true).look`, plus a `setOptions` swap. |
| 4 | df with both pair (`y min`/`y max`) AND range (`y range`) for y   | Pair wins; range stays unset.                                 |
| 5 | Tag-based detection — `column.tags['whisker.min'] = '...'` etc.   | Auto-axis-pick lands on tagged columns; whisker names match the tag values. |

## Constraints uncovered while authoring

- `ScatterPlotLook.auto(df)` runs the bulk `findMatchingErrorBarColumns`
  **only** when both `xColumnName` and `yColumnName` are unset.
  Cases 1, 2, 4, 5 therefore call `df.plot.scatter()` with no
  `{x, y}` (the original triage suggestion of `df.plot.scatter({x, y})`
  would skip the auto path entirely and the test would falsely fail).
- The per-axis `findErrorColumnsForAxis` (`core/client/d4/lib/src/utils/utils.dart`)
  is invoked from `_ScatterPlotWhiskersFeature.resetWhiskers`, which
  itself only fires on subsequent x/y prop changes when
  `_whiskersAutoDetected == true` (i.e., bulk detection already won
  during construction). For Case 5 we therefore rely on the bulk
  Phase-1 tag scan in `findMatchingErrorBarColumns`.
- Tag key convention: `findErrorColumnsForAxis` matches by
  `key.toLowerCase().contains('min' | 'max' | 'range' | 'whisker min' | ...)`.
  We use `whisker.min` / `whisker.max`; both contain `min` / `max` so
  they match the predicate.

## API surface exercised

- `df.plot.scatter()`, `DG.Viewer.scatterPlot(df, settings)`
- `IScatterPlotSettings.{xWhiskerMinColumnName, xWhiskerMaxColumnName,
   xWhiskerRangeColumnName, yWhiskerMinColumnName, yWhiskerMaxColumnName,
   yWhiskerRangeColumnName}`
- `DG.Column.tags`
- `Viewer.getOptions(true).look`, `Viewer.setOptions`

## Out of scope

- Pixel rendering of whisker bars / endcaps.
- `xWhiskerMin` / `xWhiskerMax` / `xWhiskerRange` (the non-`ColumnName`
  aliases) — these are name-binding synonyms; the canonical storage
  property is the `*ColumnName` form, which is what the test pins.
