# GROK-18223: Histogram bins range stability

Regression coverage for the histogram `bins` property. Setting a high bin
count (e.g. 1000) used to throw `RangeError (end): Invalid value: Not in
range` because the indexer was allocated before the bin count was clamped.
Fix shipped in 1.26.0 / 1.26.3 bounds the indexer up front, so neither
high values nor non-positive values throw any more.

## Triage scope

The original report: increasing the **Bins** property on the histogram
viewer to a large value blew up the viewer with a Dart `RangeError`.
Resolution clamps the indexer at allocation, so the viewer stays alive
for any numeric `bins` input. The platform may silently reject or clamp
extreme values; the contract the fix codifies is "no throw, stored value
stays a positive integer", not a specific clamp ceiling.

## Cases

| # | Case                                                                  | What it pins                                                                                |
|---|-----------------------------------------------------------------------|---------------------------------------------------------------------------------------------|
| 1 | `df.plot.histogram({value:'age'})`, sweep `setOptions({bins: n})` over `[1, 2, 50, 200, 500, 1000]` | No throw across the sweep; final stored `look.bins` is a positive finite number (exact value or clamped). |
| 2 | `setOptions({bins: 0})` then `setOptions({bins: -5})`                 | No throw; if `look.bins` is numeric it remains a positive finite integer (rejected or clamped). |
| 3 | `viewer.getProperties()` carries `bins`                               | Property found by name `bins`; `description` is a non-empty string. |

## API surface exercised

- `df.plot.histogram({value})`
- `IHistogramSettings.bins` (`d4.ts:1756`)
- `Viewer.setOptions({bins})`
- `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]`
- `Property.name`, `Property.description`

## Out of scope

- Asserting an exact clamp ceiling. The fix does not promise a specific
  upper bound, only that the viewer no longer throws.
- Visual / pixel rendering of the histogram bars.

## Risks

- `bins=0` and negative inputs may be silently rejected by Dart's typed
  setter rather than clamped. The test uses the
  "either rejected or clamped, never throws" pattern.
- `viewer.dataFrame === df` is not reliable across the Dart→JS wrapper
  boundary; we compare `rowCount` instead.
