## Triage scope

The fix exposes `showValues` on `IHistogramSettings`. We pin:

- the property key round-trips through `props['showValues']` and
  `getOptions(true).look.showValues` on every toggle;
- `getProperties()` lists the property by name with a non-empty description
  (best-effort â€” earlier `showValues` may have been a TS-only typing, so a
  missing descriptor is tolerated and the test falls back to the look-bag

- combining `showValues` with `splitColumnName` does not destabilize the
  viewer; both keys are preserved in the look bag.

Pixel-level / glyph-level rendering is out of scope for the JS API suite.
