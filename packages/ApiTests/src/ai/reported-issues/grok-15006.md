# GROK-15006: Bar chart showFilteredRows flag

**JIRA:** GROK-15006 (Done, fix in 1.19.0)
**JS API source:** `public/js-api/src/interfaces/d4.ts:872` (`IBarChartSettings.showFilteredRows`), `public/js-api/src/interfaces/d4.d.ts:82-91` (`RowSet` enum)
**Date:** 2026-04-28

## What we are testing

Regression coverage for the Bar chart `showFilteredRows` toggle that was added
in 1.19.0. The flag is documented to take effect only when `rowSource = 'All'`
(per the comment at `d4.ts:870-871`), but flipping it under other rowSource
values (e.g. `'Filtered'`) must still be a no-throw, idempotent state change.
All assertions read state via `viewer.getOptions(true).look` and `viewer.props`
— no pixel sampling, no canvas geometry.

## JS API symbols verified

- `IBarChartSettings.showFilteredRows: boolean` — `d4.ts:872`
- `IBarChartSettings.rowSource: keyof typeof RowSet` — values include `'All'`,
  `'Filtered'`, `'Selected'`, `'SelectedOrCurrent'`, `'FilteredSelected'`,
  `'MouseOverGroup'`, `'CurrentRow'`, `'MouseOverRow'` (`d4.d.ts:82-91`)
- `df.plot.bar({...})` / `DG.Viewer.barChart(df, {...})`
- `viewer.setOptions({...})`, `viewer.getOptions(includeDefaults)`,
  `viewer.props`, `viewer.getProperties()`

## Test cases

1. **`showFilteredRows=true` under `rowSource=All`** — construct
   `df.plot.bar({value:'age', split:'race'})`, then
   `setOptions({rowSource:'All', showFilteredRows: true})`. Must not throw,
   `look.rowSource === 'All'`, `look.showFilteredRows === true`.
2. **toggle `showFilteredRows` to false** — after enabling under `rowSource=All`,
   flip to false; `props.showFilteredRows === false` and `look` mirrors
   (`look.showFilteredRows === false`).
3. **`rowSource:'Filtered'` + toggle `showFilteredRows`** — under a non-`All`
   rowSource, flipping `showFilteredRows` true/false/true must still not throw
   and the final `look.showFilteredRows === true` while `look.rowSource ===
   'Filtered'`. The flag is documented as no-op visually under `Filtered` but
   is still a valid stored state.
4. **`getProperties()` lists `showFilteredRows` (best-effort)** — iterate
   `viewer.getProperties()` and assert a descriptor with `name ===
   'showFilteredRows'` is present. If the descriptor isn't exposed (as the
   triage warned), fall back to confirming it is settable via `setOptions` and
   observable via `getOptions(true).look`.

## Why state, not pixels

The fix is a flag persisted in the JSON envelope. Visual effect (showing
hatched / dimmed bars for filtered rows) is exercised Dart-side; on the JS
side we only need to confirm round-trip. We never open a `TableView` and never
call `close()` on a never-attached viewer (round-5 platform issue).

## Preconditions

- `grok.data.demo.demog(50)` is available — provides `age`, `race` columns.
- No server-side state is created. No `TableView` is opened.

## Out of scope

- Rendering of filtered-row indicator hatch / dim styling.
- Interaction with the filter group (filter UI side).
- Other `RowSet` values' interactions — only `All` and `Filtered` are
  covered, matching the triage scope.
- Trellis Bar chart inner-look propagation of `showFilteredRows`.
