# GROK-19892: Box plot horizontal slider zero range

Regression coverage for a box-plot bug fixed in 1.28.0.

## Triage scope

The horizontal slider on a box plot could be dragged so its visible range
collapsed to zero, breaking the plot. The fix prevents zero-range
("works ok on latest dev 1.28.0").

`IBoxPlotSettings` (see `js-api/src/interfaces/d4.ts:999` and the surrounding
~250 lines) does **not** expose an explicit "horizontal slider" property ã the
slider is internal viewport state. The range-y props it does expose are:

- `valueMin` / `valueMax`        ã value-axis range (vertical for default orientation)
- `colorMin` / `colorMax`        ã color scale range
- `markerMinSize` / `markerMaxSize` ã marker size mapping range

Per the triage's risk note ("If no slider state property exists in the JS
interface, fall back to a 'viewer survives extreme prop values' smoke test"),
these tests pin **state machine integrity** ã the viewer accepts matching
min/max via `setOptions`, persists them through `getOptions(true).look`, and
the instance does not throw. Pixel-level slider drag-collapse behavior belongs
in the manual / image diff suite.

## Cases

| # | Case                                                                | What it pins                                                            |
|---|---------------------------------------------------------------------|-------------------------------------------------------------------------|
| 1 | Build a box plot via `df.plot.box({value, category})`, snapshot defaults | Type is `BoxPlot`; range-y props are present and numeric-or-null.       |
| 2 | `setOptions({valueMin: 30, valueMax: 30})` (zero-range value axis)      | No throw; round-trip via `getOptions(true).look` succeeds.              |
| 3 | `setOptions({markerMinSize: 5, markerMaxSize: 5})` (zero-range markers) | No throw; `look` reflects numeric values.                               |
| 4 | `onResetView` fires when `resetView()` is called                        | Race against a 1.5s timeout (mirrors `bar-chart-js-api.ts` round-7).    |

## API surface exercised

- `df.plot.box({value, category})`
- `DG.Viewer.boxPlot(df, settings)`
- `IBoxPlotSettings.{valueMin, valueMax, colorMin, colorMax, markerMinSize, markerMaxSize, value, category1}`
- `BoxPlot.resetView()`, `BoxPlot.onResetView`
- `Viewer.setOptions`, `Viewer.getOptions(true).look`

## Out of scope

- Pixel rendering and the actual horizontal-slider drag interaction; the
  slider is not exposed via the JS API.
- We do not pin specific numeric defaults (they are platform-versioned).
- We do not call `close()` on detached viewers (only on `TableView` instances
  we attached the viewer to).
