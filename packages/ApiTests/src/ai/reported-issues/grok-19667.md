# GROK-19667: Bar chart axisType log scale stability

Regression coverage for a bar-chart bug fixed in 1.27.0.

## Triage scope

Switching `axisType` to `'logarithmic'` on certain `Split` values broke the
chart. The fix preserves viewer state under linear -> logarithmic toggles and
back, so the test pins the property surface (`IBarChartSettings.axisType`) and
its round-trip through `setOptions` / `getOptions(true).look`. The enum is
defined as:

```ts
export enum AxisType {
  linear = 'linear',
  logarithmic = 'logarithmic',
}
```

(`js-api/src/interfaces/d4.ts:769-772`), and is referenced by
`IBarChartSettings.axisType: keyof typeof AxisType` (`d4.ts:816`).

## Cases

| # | Case                                                                                  | What it pins                                                                  |
|---|---------------------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 1 | `df.plot.bar({value, split, valueAggrType: 'count'})`; `setOptions({axisType:'logarithmic'})` | No throw; `look.axisType === 'logarithmic'`.                                  |
| 2 | Switch back to `'linear'` via `setOptions`                                            | No throw; `look.axisType === 'linear'`.                                       |
| 3 | With both `valueMin` and `valueMax` set, toggle `axisType` linear <-> logarithmic     | No throw; `axisType` round-trips; range keys still echoed if platform stores them. |
| 4 | `getProperties()` includes an `axisType` property                                     | Property is present; description non-empty (best-effort, not asserted hard).  |

## API surface exercised

- `df.plot.bar({value, split, valueAggrType})`
- `DG.Viewer.barChart(df, settings)` (`js-api/src/viewer.ts:223`)
- `IBarChartSettings.axisType` (`js-api/src/interfaces/d4.ts:816`)
- `AxisType` enum (`js-api/src/interfaces/d4.ts:769-772`)
- `Viewer.setOptions`, `Viewer.getOptions(true).look`, `Viewer.getProperties()`

## Risks / unverifiable symbols

- `axisType` enum values are exact strings; case matters (`'logarithmic'`,
  not `'Logarithmic'`).
- `valueMin` / `valueMax` may not echo back through `look` if the platform
  doesn't persist them on the BarChart look — accept either outcome.
- `viewer.dataFrame === df` via strict identity is unreliable across the
  Dart -> JS wrapper boundary; we compare `rowCount` instead.
- The `axisType` property description is asserted best-effort — only
  required to be a string if present; emptiness does not fail the test.

## Out of scope

- Pixel-level rendering of a logarithmic axis or its tick layout.
- Specific defaults for `valueMin` / `valueMax` (platform-versioned).
