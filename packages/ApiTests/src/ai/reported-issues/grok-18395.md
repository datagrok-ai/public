# GROK-18395: Line chart string X-axis error

Regression coverage for a Line chart axis painter bug fixed in 1.27.0.

## Triage scope

When the X axis of a Line chart was switched from a numeric column to a
string/categorical column, `Axis.vert` threw `Unsupported operation:
NaN.round()` during paint. The fix landed on master and shipped in 1.27.0.

The bug only triggered on actual paint, so each test attaches the viewer
to a real `TableView` via `grok.shell.addTableView(df)` and waits with
`await DG.delay(300)` so the chart has a chance to repaint after the X
column swap. We do **not** subscribe to global error events — Dart errors
that hit `Axis.vert` propagate via the console only, not the JS error
hooks. The assertion is therefore a tight `expect(threw, false)` style
plus an `expect(viewer instanceof DG.LineChartViewer, true)` liveness
check after the delay.

## Cases

| # | Case                                                  | What it pins                                                                       |
|---|-------------------------------------------------------|------------------------------------------------------------------------------------|
| 1 | Numeric Y, string X — swap `xColumnName` to string    | Prop swap does not throw; viewer instance is still a `LineChartViewer` post-paint. |
| 2 | Empty string column (length 0)                        | Swapping to a zero-row string X column does not throw.                             |
| 3 | Single-value string column (all rows share one label) | Swap is safe even when the categorical X has a single category.                    |
| 4 | 1000-row mixed-string column                          | Real-world-scale categorical X column does not throw on paint.                     |

## API surface exercised

- `df.plot.line({y: 'numCol'})`
- `DG.VIEWER.LINE_CHART`, `DG.LineChartViewer`
- `viewer.props['xColumnName']` swap (numeric → string)
- `ILineChartSettings.xColumnName`
- `grok.shell.addTableView(df)` + `tv.addViewer(DG.VIEWER.LINE_CHART, ...)`

## Out of scope

- Pixel rendering, axis tick layout, label formatting. The bug surfaced
  during paint but is asserted only via "no throw" + "instance alive";
  visual correctness is not checked.
- Global Dart error capture. The original NaN.round() error is reported
  through the console; we do not hook `window.onerror` or the platform
  error stream.
