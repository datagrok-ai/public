# GROK-19760: Histogram invalid valueMin/valueMax

Regression coverage for the histogram crash on invalid `valueMin`/`valueMax`
values. Also covers GROK-19581 (same root cause). Setting `valueMin >=
valueMax` (or non-numeric) used to crash the histogram. The fix is that the
viewer survives — it either rejects or clamps the values, but it must not
throw and must keep responding to subsequent valid `setOptions` calls.

## Triage scope

The user-facing fix is defensive: invalid input does not crash the viewer.
The exact clamp / rejection policy is intentionally not pinned — any
deterministic survival is acceptable, and we only assert that the look bag
agrees with `props` after each `setOptions` call.

## Cases

| # | Case                                                            | What it pins                                                                              |
|---|-----------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| 1 | Inverted range `setOptions({valueMin: 100, valueMax: 0})`       | No throw; viewer still alive (`rowCount` matches df); look round-trips with `props`.      |
| 2 | NaN min `setOptions({valueMin: NaN, valueMax: 50})`             | No throw; viewer survives; look bag returns numeric or non-finite without breaking.       |
| 3 | Zero range `setOptions({valueMin: 50, valueMax: 50})` then sane `setOptions({valueMin: 0, valueMax: 100})` | No throw; the second, valid call lands cleanly in `look` and `props`. |
| 4 | Round-trip after each `setOptions`                              | `props['valueMin']`/`props['valueMax']` agree with `getOptions(true).look` for both keys. |

## API surface exercised

- `df.plot.histogram({value})` / `DG.Viewer.histogram`
- `IHistogramSettings.valueMin`, `IHistogramSettings.valueMax`
- `Viewer.setOptions(...)`
- `Viewer.getOptions(true).look`
- `Viewer.props[...]`

## Out of scope

- Pixel rendering / axis tick placement.
- Pinning a specific clamp value for invalid input — either rejection or
  clamp is acceptable as long as no exception escapes.
- Reading `xMin`/`xMax` (paint-time computed values, not part of `look`).

## Risks / notes

- `viewer.dataFrame === df` is not reliable across the Dart→JS wrapper
  boundary; we compare `rowCount` instead as a liveness check.
- Only assert against `getOptions(true).look`; do not rely on any
  computed-on-paint source for `xMin`/`xMax`.
