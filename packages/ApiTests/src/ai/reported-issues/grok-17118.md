# GROK-17118: Density plot binShape + axis column type

Regression coverage for fix shipped in 1.23.0. Before the fix, with
`binShape='rectangle'`, switching `xColumnName` to a string column threw.
The fix makes the density plot viewer ignore or coerce the assignment
without crashing.

## Triage scope

These tests pin the JS API contract around `binShape` and the X/Y axis
column properties on the density plot:

- `binShape='rectangle'` is accepted via `setOptions` and survives a
  `getOptions(true).look` round-trip;
- assigning a string column to `xColumnName` from rectangle state does
  not throw, the viewer instance survives (`rowCount` readable), and
  `look.xColumnName` ends up either retained as the previous numeric
  column (rejected) or set to the string column (coerced);
- a non-rectangle valid `binShape` value (discovered at runtime from
  `getProperties().find(p => p.name === 'binShape').choices`, fallback
  `'hex'`) round-trips, then switching back to `'rectangle'` stays
  throw-free;
- `getProperties()` lists `binShape`, `xColumnName`, `yColumnName` on a
  best-effort basis, with a look-bag fallback.

Visual rendering of bins, axis labels, etc. is out of scope.

## Cases

| # | Case                                                          | What it pins                                                                |
|---|---------------------------------------------------------------|-----------------------------------------------------------------------------|
| 1 | `setOptions({binShape:'rectangle'})` after `df.plot.density`  | `look.binShape === 'rectangle'`.                                            |
| 2 | From rectangle state, `setOptions({xColumnName:<stringCol>})` | No throw; viewer survives via `rowCount`; `look.xColumnName` retains old numeric or accepts string. |
| 3 | `setOptions({binShape:<other valid>})`                        | Round-trips; switch back to `'rectangle'` does not throw.                   |
| 4 | `getProperties()`                                             | Lists `binShape`, `xColumnName`, `yColumnName` (best-effort, look fallback).|

## API surface exercised

- `df.plot.density(settings)` / `DG.Viewer.densityPlot(df, settings)`
- `IDensityPlotSettings.{binShape, xColumnName, yColumnName}`
- `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`
- `Viewer.getProperties()`, `DG.Property.choices`

## Risks / notes

- `binShape` valid values are not encoded in a TS enum — the test
  discovers them at runtime via `Property.choices` and falls back to
  known literals `'hex'` / `'rectangle'`.
- `viewer.dataFrame === df` is unreliable across the Dart-JS boundary;
  we use `viewer.dataFrame?.rowCount` (or the source df's `rowCount`) as
  a survival check rather than identity-comparing dataframes.
- Some IDensityPlotSettings props are typed but not surfaced through
  `getProperties()` — case 4 falls back to a `look` round-trip.

## Out of scope

- Bin rendering, axis tick layout, color-scale visuals.
- Other `binShape` value-specific behavior beyond no-throw round-trip.
