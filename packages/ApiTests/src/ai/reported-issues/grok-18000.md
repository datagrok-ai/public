# GROK-18000: PC plot columnNames update

Regression coverage for the PC plot `columnNames` propagation fix shipped in
1.26.0/1.26.3. Removing entries from `columnNames` from the Context Panel
previously did not refresh the axes until the user manually resized the
viewer; the fix makes a `columnNames` change immediately invalidate the
layout. The JS API surface tested here is `DG.Viewer.pcPlot`
(`viewer.ts:275`) and `IPcPlotSettings.columnNames: Array<string>`
(`d4.ts:2698`), pinned through a `setOptions` round-trip into
`getOptions(true).look`.

## Triage scope

- Build a PC plot via `DG.Viewer.pcPlot(df, {columnNames: [...]})` and
  pin the initial `look.columnNames` length.
- Shrink the column set via `setOptions({columnNames: [...]})` and assert
  the look bag reflects the new array exactly тАФ no manual resize required.
- Survive degenerate transitions: empty array, then re-grow to a single
  column. Each step must update the look bag without throwing.
- Best-effort: `getProperties()` lists `columnNames` by name. Tolerated as
  a soft check because the descriptor list may not always carry it at
  runtime.

Pixel/glyph-level rendering (which axes are actually drawn) is out of scope
for the JS API suite.

## Cases

| # | Case                                                                     | What it pins                                                              |
|---|--------------------------------------------------------------------------|---------------------------------------------------------------------------|
| 1 | `DG.Viewer.pcPlot(df, {columnNames: ['age','height','weight']})`         | `look.columnNames` is an array of length 3.                               |
| 2 | `setOptions({columnNames: ['age','height']})`                            | `look.columnNames` is exactly `['age','height']` (no resize required).    |
| 3 | `setOptions({columnNames: []})` then `setOptions({columnNames: ['age']})` | Viewer survives both steps; look reflects each transition.                |
| 4 | `getProperties()` lookup for `columnNames`                                | Found by name (best-effort тАФ tolerated when descriptor is absent).        |

## API surface exercised

- `DG.Viewer.pcPlot(df, options): Viewer<IPcPlotSettings>` (`viewer.ts:275`)
- `IPcPlotSettings.columnNames: Array<string>` (`d4.ts:2698`)
- `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]` (best-effort)
- `expectArray` from `@datagrok-libraries/test/src/test`

## Risks

- `columnNames` is `Array<string>` тАФ pass real arrays, never CSV strings.
- Don't compare arrays with `===`; use `expectArray` or compare `.length`
  plus element-by-element.
- `viewer.dataFrame === df` identity is unreliable across the interop
  boundary тАФ do not assert it.
- PC plot has no `df.plot.pcPlot` shorthand. Always go through
  `DG.Viewer.pcPlot`.
