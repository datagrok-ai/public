# GROK-19511: Scatter plot linesBy independent of color

Regression coverage for improvement #3602 (Mar 2026): scatter plot acquired a
new `linesBy` / `linesByColumnName` setting so the column that splits lines
between markers is **independent** from the color column.

## Triage scope

Before the fix, lines connecting scatter plot points were split solely by the
categorical `colorColumnName`. The new property lets users pick a separate
categorical column for line grouping while leaving coloring untouched. These
tests pin the JS API contract — the new prop exists, accepts standard column
names, persists through `getOptions(true).look`, and stays decoupled from
`colorColumnName`. Visual rendering of the connecting lines is out of scope.

## Cases

| # | Case                                                                  | What it pins                                                          |
|---|-----------------------------------------------------------------------|-----------------------------------------------------------------------|
| 1 | `df.plot.scatter({x, y, color: 'race', linesBy: 'sex'})`              | Both props set, distinct values, mirrored in `look`.                  |
| 2 | `df.plot.scatter({x, y})` without `linesBy`                           | `linesByColumnName` is null or empty in both `props` and `look`.      |
| 3 | Round-trip through `getOptions(true).look.linesByColumnName`          | Value survives serialization; independence from color preserved.      |
| 4 | Mutate `linesByColumnName` via `setOptions` and direct `props[...]`   | No throw; value updates and round-trips; `colorColumnName` unchanged. |

## API surface exercised

- `df.plot.scatter({x, y, color, linesBy})`
- `DG.Viewer.scatterPlot(df, settings)`
- `IScatterPlotSettings.{linesBy, linesByColumnName, color, colorColumnName}`
- `Viewer.props[...]`, `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`

## Out of scope

- Rendering of the connecting lines, line ordering, line width.
- `linesOrder` / `linesOrderColumnName` — covered separately if needed.
- Interaction with selection or filter state.
