# GROK-19426: Pie chart showValue and showPercentage

Regression coverage for the pie-chart slice-label rename and addition shipped
in 1.27.0. `showInnerPercent` was renamed to `showPercentage`, and a new
`showValue` boolean was added next to it. The typings live in
`interfaces/d4.ts:2904-2910` (`IPieChartSettings.{showLabel, showPercentage,
showValue}`).

## Triage scope

- `props['showValue']` round-trips through `getOptions(true).look.showValue`
  on every toggle.
- `props['showPercentage']` round-trips independently — flipping one key
  must not flip the other.
- Setting both to `true` together does not throw, and both surface in the
  look bag.
- `getProperties()` lists `showValue` and `showPercentage` with non-empty
  descriptions when the runtime descriptor exposes them; otherwise the
  schema-only check (props in / look out) is the fallback.
- Soft negative on the rename: a freshly built viewer should not carry
  `showInnerPercent` in its look bag. If it lingers as a back-compat alias,
  we tolerate it — but require that `showPercentage` is the canonical
  round-trip key.

Pixel- / glyph-level rendering (whether the value is actually drawn on the
slice) is out of scope for the JS API suite.

## Cases

| # | Case                                                                    | What it pins                                                                  |
|---|-------------------------------------------------------------------------|-------------------------------------------------------------------------------|
| 1 | Build pie on `race`; toggle `props['showValue']` true→false→true         | `look.showValue` mirrors each step.                                           |
| 2 | Toggle `props['showPercentage']` while `showValue` stays at `false`      | `look.showPercentage` reflects each toggle; `look.showValue` stays `false`.   |
| 3 | `setOptions({showValue: true, showPercentage: true})`                    | No throw; both keys present in `look`.                                        |
| 4 | `getProperties()` lookup for `showValue` and `showPercentage`            | Found by name with non-empty `description`; demoted to schema-only fallback when descriptor is absent. |
| 5 | Legacy `showInnerPercent` after build and after setting `showPercentage` | Either undefined in `look`, or — if a back-compat alias survives — does not block the canonical round-trip. |

## API surface exercised

- `df.plot.pie({category})` / `DG.Viewer.pieChart` (`viewer.ts:279`)
- `IPieChartSettings.{showValue, showPercentage, showLabel}`
  (`d4.ts:2904-2910`)
- `Viewer.props[...]` (read/write)
- `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]` (best-effort)

## Out of scope

- Verifying that the value/percentage glyphs are actually drawn on the
  pie slices (DOM/canvas).
- `viewer.dataFrame === df` identity check (unreliable across the interop
  boundary).
- Strict absence of `showInnerPercent` — back-compat aliases are tolerated.
