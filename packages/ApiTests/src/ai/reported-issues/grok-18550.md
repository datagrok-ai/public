# GROK-18550: PC plot normalizeEachColumn stability

Regression coverage for the PC plot `normalizeEachColumn` toggle bug fixed in
1.26.3. Turning off Normalize Each Column with log columns set previously threw
an error. The fix lets the toggle survive in any combination with
`logColumnsColumnNames`. The JS API surface tested here is `DG.Viewer.pcPlot`
(`viewer.ts:275`), `IPcPlotSettings.normalizeEachColumn` (`d4.ts:2724`), and
`IPcPlotSettings.logColumnsColumnNames: Array<string>` (`d4.ts:2702`), pinned
through a `setOptions` round-trip into `getOptions(true).look`.

## Triage scope

- Build a PC plot via `DG.Viewer.pcPlot(df, {columnNames, logColumnsColumnNames})`
  and toggle `normalizeEachColumn` true→false→true. The viewer must not throw.
- After clearing `logColumnsColumnNames` to `[]`, toggling
  `normalizeEachColumn` repeatedly must still survive.
- Best-effort: `getProperties()` should list both `normalizeEachColumn` and
  `logColumnsColumnNames` by name. Tolerated as a soft check because the
  descriptor list may not always carry it at runtime.

Pixel/glyph-level rendering (which axis is logarithmic, etc.) is out of scope
for the JS API suite.

## Cases

| # | Case                                                                                                    | What it pins                                                                                          |
|---|---------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------|
| 1 | `pcPlot(df, {columnNames:['age','height','weight'], logColumnsColumnNames:['age']})`; flip `normalizeEachColumn` true→false→true | No throw. Final `look.normalizeEachColumn === true`. `logColumnsColumnNames` still `['age']` (or key absent). |
| 2 | `setOptions({logColumnsColumnNames:[]})`, then toggle `normalizeEachColumn` true/false repeatedly       | No throw. Final `look.normalizeEachColumn === false`. `logColumnsColumnNames` empty (or key absent).   |
| 3 | `getProperties()` lookup for `normalizeEachColumn` and `logColumnsColumnNames`                          | Found by name (best-effort — tolerated when descriptor is absent; falls back to setOptions round-trip).|

## API surface exercised

- `DG.Viewer.pcPlot(df, options): Viewer<IPcPlotSettings>` (`viewer.ts:275`)
- `IPcPlotSettings.normalizeEachColumn: boolean` (`d4.ts:2724`)
- `IPcPlotSettings.logColumnsColumnNames: Array<string>` (`d4.ts:2702`)
- `IPcPlotSettings.columnNames: Array<string>` (`d4.ts:2698`)
- `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]` (best-effort)
- `expectArray` from `@datagrok-libraries/test/src/test`

## Risks

- Empty `logColumnsColumnNames` array vs missing key on the look bag — accept
  either. Some platform builds omit empty/default arrays from serialized look.
- `logColumnsColumnNames` is `Array<string>` — pass real arrays, never CSV.
- Don't compare arrays with `===`; use `expectArray` or compare `.length` plus
  element-by-element.
- `viewer.dataFrame === df` identity is unreliable across the interop boundary;
  do not assert it.
- PC plot has no `df.plot.pcPlot` shorthand. Always go through
  `DG.Viewer.pcPlot`.
