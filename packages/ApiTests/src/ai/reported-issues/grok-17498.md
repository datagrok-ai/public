# GROK-17498: Pie chart includeNulls property

Regression coverage for the pie chart `includeNulls` toggle. The triage
reports the flag had behaved erratically; the fix is a stable property
round-trip for the null-bucket inclusion. The typing lives at
`IPieChartSettings.includeNulls` (`d4.ts:2866`).

## Triage scope

- the `includeNulls` key round-trips through `props['includeNulls']` and
  `getOptions(true).look.includeNulls` on every toggle, with a small
  hand-built DataFrame that has explicit DG missing values in the category
  column (verified via `column.stats.missingValueCount > 0`);
- with `includeNulls = true` the viewer instance survives a subsequent
  `setOptions({categoryColumnName: <other>})`; both `includeNulls` and the
  new `categoryColumnName` remain coherent in the look bag;
- `getProperties()` lists `includeNulls` by name with a non-empty
  description (best-effort — if the runtime descriptor list does not carry
  it, we fall back to the schema-only check we already exercised).

Pixel-level rendering (whether the null bucket is actually drawn as a
slice) is out of scope for the JS API suite.

## Cases

| # | Case                                                                               | What it pins                                                                       |
|---|------------------------------------------------------------------------------------|------------------------------------------------------------------------------------|
| 1 | Build df with `Column.fromList('string', 'cat', ['A', null, 'B', null, 'A'])`      | Column reports `stats.missingValueCount > 0` (true DG missing values, not literal `'null'`). |
| 2 | `DG.Viewer.pieChart(df, {category:'cat'})`; toggle `includeNulls` true→false→true  | `look.includeNulls` mirrors each step.                                              |
| 3 | With `includeNulls=true`, `setOptions({categoryColumnName:<other>})`               | Viewer survives; `look.includeNulls` and `look.categoryColumnName` both preserved.  |
| 4 | `getProperties()` lookup for `includeNulls`                                        | Found by name; `description` non-empty when listed (best-effort).                   |

## API surface exercised

- `DG.Column.fromList('string', name, values)` with embedded JS `null`
- `DG.DataFrame.fromColumns([...])`
- `column.stats.missingValueCount`
- `DG.Viewer.pieChart(df, {category, categoryColumnName})`
- `IPieChartSettings.includeNulls`, `IPieChartSettings.categoryColumnName`
- `Viewer.props[...]` (read/write)
- `Viewer.setOptions(...)`, `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]` (best-effort)

## Out of scope

- Whether the null bucket is rendered as a slice (canvas/DOM).
- `viewer.dataFrame === df` identity (unreliable across the interop boundary).

## Risks

- "Null" semantics — the test must use a true DG missing value, not the
  literal string `'null'`. We assert `stats.missingValueCount > 0` on the
  built column; if `Column.fromList` happens to coerce JS `null` to the
  string `'null'` on a future client, the assertion fires and the suite
  falls back to `grok.data.demo.demog(50)` (which has nulls in some
  columns) — handled inline.
- The triage said `df.plot.pie(...)`; the actual plot helper exposes no
  `pie` shorthand, so we use the documented `DG.Viewer.pieChart(df, ...)`
  factory instead (matches the existing PieChart suite).
