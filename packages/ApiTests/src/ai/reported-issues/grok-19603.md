# GROK-19603: Histogram show distribution lines

Regression coverage for the new histogram per-category distribution lines
feature. When `splitStack` is on with bins displayed, the histogram can
overlay per-category distribution lines via the new `showDistributionLines`
flag (verified at `js-api/src/interfaces/d4.ts:1822`).

## Triage scope

The user-facing change adds a new boolean property `showDistributionLines`
to `IHistogramSettings`. Visible drawing differences (whether/how lines are
actually painted) are out of scope for the JS API suite — assert the
state-machine only.

## Cases

| # | Case                                                                       | What it pins                                                                                  |
|---|----------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------|
| 1 | `df.plot.histogram({value, splitColumnName})`, `splitStack=true`, toggle the flag true/false twice | `props['showDistributionLines']` and `look.showDistributionLines` agree on each toggle.       |
| 2 | Without a split (`splitColumnName=''`), toggle the flag                    | Round-trip survives an empty split; no exceptions thrown; `look` mirrors final state.         |
| 3 | `viewer.getProperties()` exposes the property                              | Found by name `showDistributionLines`; description checked best-effort, with look fallback.   |

## API surface exercised

- `df.plot.histogram({value, splitColumnName})`
- `IHistogramSettings.showDistributionLines`
- `IHistogramSettings.splitStack`
- `IHistogramSettings.splitColumnName`
- `Viewer.props[...]` (read/write)
- `Viewer.getOptions(true).look`
- `Viewer.getProperties(): Property[]`
- `Property.name`, `Property.description`

## Out of scope

- Verifying that the distribution lines are actually rendered on the canvas.
- Visual / pixel-level assertions on the histogram.

## Risks

- `getProperties()` may not enumerate newly-added props in older descriptors;
  the third case demotes to a look-bag round-trip in that case so the
  regression remains meaningful.
