# Chemspace

Datagrok plugin that integrates the [Chemspace](https://chem-space.com) compound catalog: structure / text search, contextual info panels for a selected molecule and for a `chemspace-id` cell, and "Add new column" helpers that enrich a dataframe with Chemspace ids and prices.

## Where to look

| Concern | File |
|---|---|
| All registered functions (app, panels, vector funcs, HTTP helper) | [src/package.ts](src/package.ts) |
| Chemspace REST response types + flattened grid row types | [src/model.ts](src/model.ts) |
| Country display name → ISO-2 code map used by every `shipToCountry` param | [src/country-codes.ts](src/country-codes.ts) |
| `chemspace-id` regex (mirrors `meta.semanticTypes` in package.json) | [detectors.js](detectors.js) |

| Custom styles (`chemspace-*` classes referenced from package.ts) | [css/chemspace.css](css/chemspace.css) |

All public-facing functions are static methods of `PackageFunctions` in [src/package.ts](src/package.ts), annotated with `@grok.decorators.*` (the newer decorator-based registration, not `//name:` comments). `package.g.ts` and `package-api.ts` are regenerated by `grok api`.

## Glossary

- **Chemspace** — Enamine's aggregated catalog. REST API at `https://api.chem-space.com/v4/`. Access is via a bearer `access_token` obtained by exchanging a long-lived `apiKey` at `GET /auth/token`. `apiKey` lives in Datagrok package credentials under parameter name `apiKey`.
- **Category** — result classes the API segments on: `CSMS`, `CSMB`, `CSCS`, `CSSB`, `CSSS` (make-on-demand vs. in-stock × screening vs. building-blocks, plus synthesis services). Enum `CATEGORY` in [src/package.ts](src/package.ts).
- **Search mode** — `exact` / `sim` / `sub` / `text`. User-visible names live in `SEARCH_MODE`; `modeToParam` maps them to the URL segment. App offers all four; `samplesPanel` uses Similar + Substructure; `pricesPanel` and `getChemspacePrices` use `text`; `getChemspaceIds` uses `exact`.
- **`chemspace-id`** — semantic type for a Chemspace compound id. Regex `^(CSMS|CSMB|CSCS|CSSB|CSSS)\d{11}$`. Declared in `package.json` `meta.semanticTypes` **and** [detectors.js](detectors.js); `pricesPanel` is bound to columns of this semType.
- **offer / price** — one `ChemspaceResult` has many `offers` (one per vendor), each with many `prices` (one per pack size in mg).

## Conventions specific to this package

- **All external HTTP goes through `queryMultipart`.** Higher-level functions call it via `grok.functions.call(\`${_package.name}:queryMultipart\`, …)` rather than importing it, so the function-level cache (`meta.cache: all`, invalidated monthly via `0 0 1 * *`) deduplicates identical searches across the app, panels, and vector funcs. Do not add a second HTTP path.
- **Token** is a module-level singleton (`let token`). On HTTP 401 it is cleared and refetched; on 429 the helper retries up to 10 times with linearly growing delays. `_package.getCredentials()` reads `apiKey` from package credentials.
- **Country inputs** are shown to the user by display name (key of `COUNTRY_CODES`) and converted to ISO-2 at the API call site — never pass the raw display name to the Chemspace API.
- **`samplesPanel` ranks Similar hits client-side** with `grok.chem.findSimilar` on the SMILES column returned by the server, so top-20 ordering reflects RDKit similarity, not server order.
- `getChemspacePrices` is declared with `outputs: [{… options: {action: 'join(data)'}}]` so the resulting dataframe is joined back into the caller's `data`. Because it joins, it must produce exactly one row per input id — `selectFirstAvailableOffer` collapses each `ChemspaceResult` to the first offer that has at least one price (and that offer's first price entry). If you need per-pack-size or per-vendor rows, build a separate function rather than widening this one.
