## Directory Purposes

**`src/analysis/`:**
- Purpose: Domain computation — normalization, imputation, DE, PCA, enrichment
- Contains: Pure functions operating on `DG.DataFrame`; dialog launchers (`showX()`)
- Key files: `differential-expression.ts` (limma/DEqMS/t-test pipeline), `enrichment.ts` (g:Profiler integration), `experiment-setup.ts` (group management)

**`src/parsers/`:**
- Purpose: Convert raw file bytes/text into semantically-typed, filtered `DG.DataFrame`
- Contains: Parser functions + shared helpers for log2 transform, delimiter detection, column suggestion
- Key files: `maxquant-parser.ts`, `spectronaut-parser.ts`, `generic-parser.ts`, `shared-utils.ts`

**`src/viewers/`:**
- Purpose: Create `DG.Viewer` instances from an already-analysed DataFrame
- Contains: Viewer factory functions (`createX()`, `openX()`); QC computations
- Key files: `volcano.ts`, `heatmap.ts`, `pca-plot.ts`, `qc-dashboard.ts`, `qc-computations.ts`, `enrichment-viewers.ts`

**`src/panels/`:**
- Purpose: Context panel widgets registered with the platform
- Contains: `uniprot-panel.ts` — fetches protein metadata from UniProt REST API

**`src/utils/`:**
- Purpose: Cross-cutting constants and helpers imported by all other layers
- Key files: `proteomics-types.ts` (SEMTYPE constants), `column-detection.ts` (`findColumn`, `findProteomicsColumns`)

**`src/tests/`:**
- Purpose: Test suites imported by `src/package-test.ts`
- Contains: One file per domain area (parsers, analysis, QC, enrichment, Spectronaut)

**`scripts/`:**
- Purpose: Server-side R scripts registered as platform functions via `#name:` / `#language: r` metadata
- Contains: `limma_de.R`, `deqms_de.R`, `vsn_normalize.R`
- Each declares a conda environment with required Bioconductor packages

**`files/demo/`:**
- Purpose: Demo data files loaded by `proteomicsDemo()` via `_package.files.readAsText('demo/...')`
- Files: `proteinGroups.txt` (MaxQuant), `cptac-spike-in.txt` (CPTAC benchmark), `spectronaut-hye-mix.tsv` (Spectronaut)

**`dist/`:**
- Purpose: Webpack output — `package.js` (main bundle) and `package-test.js` (test bundle)
- Generated: Yes
- Source-controlled: No (included in `.gitignore`)

**`.planning/`:**
- Purpose: GSD workflow planning tree — codebase maps, milestone phases, research notes, todos
- Generated: No (hand-maintained)
- Source-controlled: Yes

## Key File Locations

**Entry Points:**
- `src/package.ts`: Main entry — `PackageFunctions` class, all `@grok.decorators.*` registrations, top-menu menu tree
- `src/package-test.ts`: Test entry — imports all `src/tests/*.ts` files, exports `tests` array for `grok test`
- `detectors.js`: Semantic type detectors — plain JS, loaded by platform before the webpack bundle

**Configuration:**
- `webpack.config.js`: Two entry points (`src/package.ts`, `src/package-test.ts`); externals `DG`, `grok`, `ui`, `rxjs`; output to `dist/`
- `tsconfig.json`: ES2020 target, strict mode, decorators enabled
- `package.json`: `@datagrok/proteomics` v0.1.0; key deps: `@datagrok-libraries/statistics`, `@datagrok-libraries/bio`, `datagrok-api`

**Core Logic:**
- `src/utils/proteomics-types.ts`: All SEMTYPE string constants — single source of truth for semantic type values
- `src/utils/column-detection.ts`: `findColumn()` and `findProteomicsColumns()` — used everywhere to locate DataFrame columns
- `src/parsers/shared-utils.ts`: `log2TransformColumns`, `copyAsLog2Columns`, `detectLog2Status`, `addPrimaryColumnIfNeeded`, `detectDelimiter`
- `src/analysis/experiment-setup.ts`: `GroupAssignment` interface; `getGroups()` / `setGroups()` — serializes workflow state into `proteomics.groups` tag

**Auto-Generated (do not edit):**
- `src/package.g.ts`: Generated by `grok api` from `@grok.decorators.*` calls in `src/package.ts`
- `src/package-api.ts`: Generated typed wrappers; regenerated on every `npm run build`

## Naming Conventions

**Files:**
- kebab-case: `differential-expression.ts`, `maxquant-parser.ts`, `qc-dashboard.ts`
- No file-type suffix in the name (not `maxquant-parser.service.ts`)

**Directories:**
- Lowercase with no hyphens: `analysis/`, `parsers/`, `viewers/`, `panels/`, `utils/`, `tests/`

**Functions:**
- Dialog launchers: `showX(df)` — opens a `ui.dialog(...)`, e.g., `showDEDialog`, `showNormalizationDialog`
- View launchers: `openX(df)` — creates table view or dashboard, e.g., `openQcDashboard`, `openEnrichmentVisualization`
- Viewer constructors: `createX(df, options?)` — returns a `DG.Viewer` without side effects, e.g., `createVolcanoPlot`, `createExpressionHeatmap`, `createPcaPlot`
- Parser functions: `parseXText(text)` (sync) and `parseX(file)` (async wrapper), e.g., `parseMaxQuantText`
- Computation functions: verb + noun, e.g., `medianNormalize`, `imputeKnn`, `runDifferentialExpression`, `computePCA`

**Classes:**
- PascalCase: `PackageFunctions`, `ProteomicsPackageDetectors`

**Constants:**
- SCREAMING_SNAKE_CASE object with `as const`: `SEMTYPE.PROTEIN_ID`, `SEMTYPE.GENE_SYMBOL`

## Where to Add New Code

**New parser (e.g., FragPipe, DIA-NN):**
- Parser function: `src/parsers/<format>-parser.ts` — export `parse<Format>Text(text): DG.DataFrame`
- Register: Add `import` + `@grok.decorators.func({'top-menu': 'Proteomics | Import | <Format>...'})` handler in `src/package.ts`
- Tests: `src/tests/parsers.ts` or a new `src/tests/<format>-parser.ts` imported in `src/package-test.ts`

**New analysis step:**
- Implementation: `src/analysis/<step-name>.ts` — export `show<Step>Dialog(df)` and the core computation function separately for testability
- Register: Add handler in `src/package.ts` with precondition tag check
- State: Set a new `df.setTag('proteomics.<step>', 'true')` on completion
- Tests: `src/tests/analysis.ts` (extend) or new `src/tests/<step-name>.ts`

**New viewer:**
- Implementation: `src/viewers/<viewer-name>.ts` — export `create<Viewer>(df, options?)` returning `DG.Viewer`
- Register: Add `@grok.decorators.func({'top-menu': 'Proteomics | Visualize | <Viewer>...'})` handler in `src/package.ts` with required tag precondition check

**New R script:**
- Script: `scripts/<name>.R` with `#name:`, `#language: r`, `#environment:`, `#input:`, `#output:` headers
- Invocation: `await grok.functions.call('Proteomics:<FunctionName>', {param: value})` from TypeScript
- Always add a client-side fallback in the calling TypeScript code

**New utility:**
- Column helpers: `src/utils/column-detection.ts`
- Type constants: `src/utils/proteomics-types.ts`
- Parser-only helpers: `src/parsers/shared-utils.ts`

**New demo data:**
- Place files in `files/demo/`
- Load via `await _package.files.readAsText('demo/<filename>')`

## Special Directories

**`dist/`:**
- Purpose: Webpack bundle output (`package.js`, `package-test.js`, source maps)
- Generated: Yes (by `npm run build` / webpack)
- Source-controlled: No

**`detectors.js` (root file, not a directory):**
- Purpose: Semantic type detectors loaded by platform before the bundle; plain JS, no TypeScript
- The platform loads this file independently, so it must not use ES module `import` syntax
- Defines `ProteomicsPackageDetectors extends DG.Package` with `//meta.role: semTypeDetector` functions

**`.planning/`:**
- Purpose: GSD project planning documents
- Contains: `codebase/` (architecture maps), `milestones/v1.0-phases/` through `v1.3-phases/` (completed and in-progress phase plans), `research/`, `todos/pending/`, `todos/done/`, `debug/`
- Source-controlled: Yes
