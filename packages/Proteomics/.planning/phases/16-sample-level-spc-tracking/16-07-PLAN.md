```
export function createParetoChart(
  runsDf: DG.DataFrame,            // runs slice DataFrame from openSpcDashboard (sibling TableView df)
  instrumentId: string,
): DG.Viewer | null;                // returns null when slice has < 4 runs (Pareto needs ≥ 4)
```

Aggregation logic:
- For each row in runsDf, parse `rules_tripped` (JSON-stringified string[]) — e.g. `["nelson_1@median_intensity", "nelson_5@control_corr"]`.
- Build a flat list of `(rule_metric_key, run_id)` pairs, then group by `rule_metric_key` to get trip counts.
- Result DataFrame columns: `rule_metric` (string, e.g. `'nelson_1@median_intensity'`), `trip_count` (int), `total_runs` (int, same across rows = runsDf.rowCount), `pct` (float, `trip_count / total_runs * 100`).
- Sort by `trip_count` DESCENDING.

DG.Viewer.barChart options:
- `splitColumnName: 'rule_metric'`
- `valueColumnName: 'trip_count'`
- `title: \`Which rules trip most often — \${instrumentId}\``
- DG barChart honors descending bar order when the underlying DF is pre-sorted; set the result DF's sort BEFORE constructing the viewer.

From `datagrok-api/dg`:
- `DG.Viewer.barChart(df, options)`
- `DG.DataFrame.fromColumns([DG.Column.fromStrings(...), DG.Column.fromInt32Array(...), ...])`

UI-SPEC copy (verbatim):
- X-axis label: `(rule, metric)`
- Y-axis label: `Times tripped`
- Chart title: `Which rules trip most often — <instrument_id>`
- Bar tooltip: `<rule_id>@<metric>: <N> trips across <M> runs (<pct>%)`
- Empty-state body (when slice.length < 4): `Pareto chart appears once 4 or more runs have been recorded for this instrument.`
