  <files>packages/Proteomics/src/viewers/qc-computations.ts</files>
  <action>
Create `qc-computations.ts` with the following pure computation functions (no UI/viewer code):

**`getIntensityColumns(df: DG.DataFrame): string[]`**
- Return column names where `semType === SEMTYPE.INTENSITY && name.startsWith('log2(')`.

**`computeMA(df: DG.DataFrame, groups: GroupAssignment): void`**
- For each protein row, compute M = mean(group2 log2 intensities) - mean(group1 log2 intensities), A = (mean(g1) + mean(g2)) / 2.
- Add `M` and `A` float columns to df. If columns already exist, remove and recreate.
- Guard: if all values in a group are null for a row, set M and A to `DG.FLOAT_NULL`.
- Helper: `groupMean(cols: DG.Column[], rowIdx: number): number` -- returns NaN if count is 0.

**`computeLoessTrend(df: DG.DataFrame, windowFraction: number = 0.1): void`**
- Compute a moving-average trend line for the MA plot (per CONTEXT.md locked decision: "Include loess/lowess trend line showing systematic intensity-dependent bias"). Implementation approach is at Claude's discretion per CONTEXT.md.
- Sort rows by A values. Use a sliding window of size `Math.max(Math.round(rowCount * windowFraction), 10)` centered on each point.
- For each point, compute the mean M value within the window (ignoring DG.FLOAT_NULL values in both M and A).
- Add a float column `MA_trend` to df. If it already exists, remove and recreate.
- Guard: if fewer than 3 non-null A values, skip trend computation and set all MA_trend values to DG.FLOAT_NULL.
- This is a simple moving-average approximation of loess -- sufficient for showing systematic intensity-dependent bias patterns. A proper loess implementation would require weighted local regression, which is unnecessary complexity for a QC overview.

**`computeCV(df: DG.DataFrame, groupCols: string[], cvColName: string, meanColName: string): void`**
- For each protein row, compute CV = sd / mean on RAW (non-log) intensities within the group. Exponentiate log2 values: `Math.pow(2, log2Val)`.
- Add float columns `cvColName` and `meanColName` to df. Remove and recreate if they exist.
- Guard: need count >= 2 for CV, count >= 1 for mean. Use `DG.FLOAT_NULL` otherwise.
- Use sample variance: `(sumSq - sum^2/count) / (count - 1)`.

**`createMissingnessMatrix(df: DG.DataFrame, intensityCols: string[]): DG.DataFrame`**
- Create a new DataFrame where each intensity column becomes a binary int column: 1 = value present, 0 = missing.
- Include protein ID column (clone from original using `findColumn(df, SEMTYPE.PROTEIN_ID, ['protein id', 'accession'])`).
- Name the returned DataFrame `'Missing Values'`.

**`unpivotIntensities(df: DG.DataFrame, intensityCols: string[]): DG.DataFrame`**
- Create long-format DataFrame with columns: `ProteinId` (string), `Sample` (string), `Intensity` (float).
- For each protein row x intensity column, add a row (skip null intensity values to keep DataFrame compact).
- Get protein ID from the protein ID column in df. If no protein ID column, use row index as string.
- Name the returned DataFrame `'Intensity Distributions'`.

**`computeMissingBarData(df: DG.DataFrame, intensityCols: string[], groups: GroupAssignment): DG.DataFrame`**
- Create a sample-level DataFrame with columns: `Sample` (string), `MissingPct` (float), `Group` (string).
- For each intensity column, count null rows / total rows * 100.
- Assign group name by checking if the column name is in group1.columns or group2.columns.
- Name the returned DataFrame `'Missing Values per Sample'`.

Import `DG` from `datagrok-api/dg`, `SEMTYPE` from `../utils/proteomics-types`, `findColumn` from `../utils/column-detection`, and `GroupAssignment` type from `../analysis/experiment-setup`.

<task type="auto">
  <name>Task 2: Create QC dashboard orchestration with viewer layout</name>
  <files>packages/Proteomics/src/viewers/qc-dashboard.ts</files>
  <action>
Create `qc-dashboard.ts` that orchestrates all QC viewers into a docked layout.

**`openQcDashboard(df: DG.DataFrame): void`**

1. **Prerequisite check:** Call `getGroups(df)`. If null, show `grok.shell.warning('Annotate experimental groups first (Proteomics | Annotate Experiment)')` and return. Same pattern as PCA in package.ts.

2. **Clean up previous QC columns/viewers:** Remove columns M, A, MA_trend, CV_group1, meanInt_group1, CV_group2, meanInt_group2 if they exist (allows re-running). Use `df.columns.contains(name) && df.columns.remove(name)` pattern.

3. **Get intensity columns:** Call `getIntensityColumns(df)`. If empty, warn and return.

4. **Compute MA values:** Call `computeMA(df, groups)`.

5. **Compute loess/moving-average trend:** Call `computeLoessTrend(df)`. This adds the `MA_trend` column used to overlay the trend line on the MA scatter plot per CONTEXT.md locked decision.

6. **Compute CV per group:** Call `computeCV(df, groups.group1.columns, 'CV_group1', 'meanInt_group1')` and same for group2.

7. **Create MA scatter plot:** `tv.addViewer(DG.VIEWER.SCATTER_PLOT, {x: 'A', y: 'M', title: 'MA Plot'})`. Add horizontal line at M=0 using `markerDefaultSize: 2` for clean look. If `df.getTag('proteomics.de_complete') === 'true'`, call `ensureDirectionColumn(df, 1.0, 0.05)` and set `colorColumnName: 'direction'` on the scatter plot. Otherwise, no color column.

    **MA trend line overlay:** Add a second scatter plot series or a line viewer showing the trend. Approach options (use whichever works with the Datagrok API):
    - Option A: Add a second `DG.VIEWER.SCATTER_PLOT` on the same DataFrame with `{x: 'A', y: 'MA_trend', markerType: 'line'}` and dock it overlaid/alongside the MA scatter. This is the simplest approach.
    - Option B: If `SCATTER_PLOT` supports a `showRegressionLine` or similar property, use that.
    - Option C: Create a line chart viewer (`DG.VIEWER.LINE_CHART`) with A on x-axis and MA_trend on y-axis, docked with/near the MA scatter.
    The key requirement is that the trend line is visible alongside the MA scatter data so scientists can see systematic intensity-dependent bias. If none of the overlay approaches work cleanly, add the trend as a separate small viewer docked directly below the MA plot.

8. **Create CV scatter plot:** `tv.addViewer(DG.VIEWER.SCATTER_PLOT, {x: 'meanInt_group1', y: 'CV_group1', title: 'CV Plot'})`. Use group1 CV by default.

9. **Create sample correlation plot:** `tv.addViewer(DG.VIEWER.CORR_PLOT, {xColumnNames: intensityCols, yColumnNames: intensityCols})`. Set title `'Sample Correlation'`.

10. **Create missing values heatmap:** Call `createMissingnessMatrix(df, intensityCols)`. Register the DataFrame with `grok.shell.addTable(missDf)` (not addTableView -- keep it headless). Create a grid viewer on the missingness DataFrame. Use `tv.addViewer('Grid', {dataFrame: missDf, ...})` or add a separate TableView if needed (follow heatmap.ts clone pattern). Title: `'Missing Values'`.

11. **Create missing values bar chart:** Call `computeMissingBarData(df, intensityCols, groups)`. Register with `grok.shell.addTable(barDf)`. Create a bar chart viewer: `DG.VIEWER.BAR_CHART` with `{split: 'Sample', value: 'MissingPct', valueAggrType: 'avg'}`. Color by Group column. Title: `'Missing % per Sample'`.

12. **Create box plot for intensity distributions:** Call `unpivotIntensities(df, intensityCols)`. Register with `grok.shell.addTable(longDf)`. Create: `DG.VIEWER.BOX_PLOT` with `{value: 'Intensity', category1ColumnName: 'Sample'}`. Title: `'Intensity Distributions'`.

13. **Dock layout:** Use `tv.dockManager.dock()` to arrange viewers in a roughly 2x3 or 3x2 grid. Suggested approach:
    - MA plot docked RIGHT of default grid (ratio 0.5)
    - CV plot docked DOWN from MA plot (ratio 0.5)
    - Correlation plot docked RIGHT of MA/CV column (ratio 0.33)
    - Missing heatmap, missing bar, and box plot can be docked DOWN from the main area

    The exact layout may need adjustment -- use reasonable ratios. The key constraint is all viewers visible simultaneously without scrolling.

14. **Show progress:** Wrap computation in `DG.TaskBarProgressIndicator.create('Creating QC dashboard...')` with try/finally close.

Import from: `datagrok-api/grok`, `datagrok-api/ui`, `datagrok-api/dg`, `./qc-computations` (all computation functions including computeLoessTrend), `../analysis/experiment-setup` (getGroups, GroupAssignment), `./volcano` (ensureDirectionColumn).

**Selection behavior note:** MA scatter and CV scatter share the main DataFrame, so selecting proteins in one highlights them in the other. The missing values heatmap, missing bar chart, and box plot use SEPARATE DataFrames -- cross-selection with protein-level viewers will NOT work for these. This is acceptable per CONTEXT.md decisions. Do not attempt workarounds for cross-DataFrame selection.

Note on the missing values heatmap: Since it uses a separate DataFrame, the simplest approach is to create a separate TableView for it (like PCA does for sample-level data) or add it as a viewer in the current TV referencing the separate DataFrame. Try the `tv.addViewer()` approach first; if it requires same-DataFrame, use a separate docked panel or skip the heatmap grid and rely on the bar chart for missing value visualization.
