  <action>
Create `src/parsers/shared-utils.ts` with these exported functions:

1. **`log2TransformColumns(df, colNames)`** -- For each column name: set col.semType = SEMTYPE.INTENSITY, create `log2(name)` float column with Math.log2 values (DG.FLOAT_NULL for zero/negative/null), set log2Col.semType = SEMTYPE.INTENSITY. Extract from maxquant-parser.ts lines 90-125, but accept explicit column names instead of auto-detecting by prefix.

2. **`copyAsLog2Columns(df, colNames)`** -- Same as above but copies values as-is (no Math.log2). For already-log2-transformed data. Sets both original and log2 column semType = SEMTYPE.INTENSITY.

3. **`addPrimaryColumnIfNeeded(df, sourceColName, newName, semType)`** -- Scan source column for semicolons. If found, create primary column with first entry. If no semicolons, skip creation entirely. Extract from maxquant-parser.ts lines 54-68, adding the semicolon-detection guard per CONTEXT.md decision.

4. **`detectLog2Status(df, colNames)`** -- Returns `{isLog2: boolean, message: string}`. Sample up to 200 non-null numeric values from the specified columns. If >50% are >= 1000, return `{isLog2: false, message: 'Data appears to be raw intensities'}`. If >80% are in [0, 30], return `{isLog2: true, message: 'Data appears to be already log2-transformed'}`. Default: `{isLog2: false, message: 'Could not determine data scale'}`. See RESEARCH.md Pattern 3 for reference implementation.

5. **`detectDelimiter(text)`** -- Returns '\t' or ','. Count tabs vs commas in first line. See RESEARCH.md Pattern 5.

6. **`autoSuggestProteinIdColumn(df)`** -- Returns `DG.Column | null`. Scan string columns for names containing (case-insensitive): 'protein', 'accession', 'uniprot'. Return first match.

7. **`autoSuggestIntensityColumns(df)`** -- Returns `string[]`. Scan numeric column names for keywords (case-insensitive): 'intensity', 'lfq', 'ibaq', 'tmt', 'reporter', 'abundance'. Return all matching names.

Then refactor `maxquant-parser.ts`:
- Import `log2TransformColumns` and `addPrimaryColumnIfNeeded` from `./shared-utils`.
- Replace `processIntensityColumns()` body: still auto-detect intensity columns by prefix (MaxQuant-specific logic stays), but call `log2TransformColumns(df, detectedNames)` for the actual transform.
- Replace `addPrimaryColumn()` calls with `addPrimaryColumnIfNeeded()` calls, passing the specific source column name found by `findCol()`.
- Keep `buildIntensityColumnOptions()`, `findCol()`, `filterByMarker()`, `filterByIdPrefix()`, `assignSemanticTypes()` in maxquant-parser.ts -- they are MaxQuant-specific.
- Ensure the public API (`parseMaxQuantText`, `parseMaxQuant`) is unchanged.
