  <files>packages/Proteomics/src/parsers/generic-parser.ts</files>
  <action>
Create `src/parsers/generic-parser.ts` exporting `showGenericImportDialog()`.

**File loading:** Use `DG.Utils.openFile()` to let user pick a CSV/TSV file:
1. Call `DG.Utils.openFile({accept: '.csv,.tsv,.txt'})` to get file text.
2. Call `detectDelimiter(text)` and parse with `DG.DataFrame.fromCsv(text, {delimiter})`.
3. Show the column mapping dialog with the parsed DataFrame.

**Dialog construction** (single dialog, all fields visible at once):

```
ui.dialog('Import Generic Matrix')
  .add(proteinIdInput)        // Single column dropdown, string columns only
  .add(geneNameInput)         // Single column dropdown, string columns only, optional
  .add(intensityColsInput)    // Multi-select columns input
  .add(log2Toggle)            // Boolean toggle
  .add(hintDiv)               // Dynamic hint message about log2 detection
  .add(previewContainer)      // Grid showing first 5 rows of selected columns
  .addButton('Import', importHandler)
  .show();
```

**Protein ID input:** `ui.input.column('Protein ID', {table: df, filter: (c) => c.type === DG.COLUMN_TYPE.STRING, value: autoSuggestProteinIdColumn(df)})`. Auto-suggest using shared-utils function.

**Gene Name input:** `ui.input.column('Gene Name (optional)', {table: df, filter: (c) => c.type === DG.COLUMN_TYPE.STRING, value: null, nullable: true})`. No auto-suggestion -- purely optional.

**Intensity Columns input:** `ui.input.columns('Intensity Columns', {table: df, available: suggestedNames, value: suggestedCols})` where `suggestedNames = autoSuggestIntensityColumns(df)` and `suggestedCols` are the actual DG.Column objects for those names.

**Log2 Toggle:** Call `detectLog2Status(df, intensityColNames)` to get initial state. Create `ui.input.bool('Log2 Transform', {value: !detectedIsLog2})`. When intensity column selection changes, re-run detection and update toggle + hint.

**Hint div:** A `ui.div()` displaying the detection message (e.g., "Data appears to be raw intensities"). Style with `ui.element('div')` or `document.createElement('div')` with italic/gray text. Update on intensity column change and toggle change.

**Live preview:** Create a container div with fixed height (200px, overflow: auto). On every column selection change:
- Collect selected column names: [proteinId, geneName (if set), ...intensityCols] (first 5 intensity cols max for preview width)
- Clone first 5 rows of df with only selected columns: `df.clone(DG.BitSet.create(df.rowCount, (i) => i < 5), selectedColNames)`
- Create grid: `DG.Viewer.grid(previewDf).root`
- Replace container children with new grid root

**Import handler (onOK):**
1. Get selected columns from inputs.
2. Validate: protein ID must be selected, at least 1 intensity column must be selected.
3. Assign `proteinIdCol.semType = SEMTYPE.PROTEIN_ID`.
4. If gene name column selected: assign `geneNameCol.semType = SEMTYPE.GENE_SYMBOL`.
5. If log2Toggle is ON (needs transform): call `log2TransformColumns(df, intensityColNames)`.
6. If log2Toggle is OFF (already log2): call `copyAsLog2Columns(df, intensityColNames)`.
7. Call `addPrimaryColumnIfNeeded(df, proteinIdCol.name, 'Primary Protein ID', SEMTYPE.PROTEIN_ID)`.
8. If gene name selected: call `addPrimaryColumnIfNeeded(df, geneNameCol.name, 'Primary Gene Name', SEMTYPE.GENE_SYMBOL)`.
9. Set `df.setTag('proteomics.source', 'generic')`.
10. Set `df.name` to filename without extension (if from file) or keep existing name (if from table).
11. Call `grok.shell.addTableView(df)`.
12. Call `grok.shell.info('Imported N proteins from filename')`.

**Error handling:** Wrap file parsing in try/catch. If `fromCsv` fails, show `grok.shell.error('Failed to parse file: ' + e.message)`. If no numeric columns found for intensity, show warning in dialog.

**Important:** Do NOT use `dialog.onOK()` for the import button -- use `dialog.addButton('Import', handler)` so that the "Import" label is explicit per CONTEXT.md decision ("Import and Cancel buttons at bottom"). Use `dialog.addButton('Cancel', () => dialog.close())` or rely on the dialog's built-in close behavior.

Actually, `ui.dialog().onOK()` already creates an OK button. Rename it by passing the label: check if `ui.dialog` supports button label customization. If not, use the default OK button via `.onOK(importHandler)` -- the platform handles Cancel automatically.
