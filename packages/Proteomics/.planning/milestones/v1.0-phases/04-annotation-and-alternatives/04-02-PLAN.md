  <name>Task 1: Create DEqMS R script with same output contract as limma</name>
  <files>packages/Proteomics/scripts/deqms_de.R</files>
  <action>
Create `scripts/deqms_de.R` with the following:

1. **Metadata header** (same pattern as limma_de.R):
   ```r
   #name: deqmsDE
   #description: DEqMS peptide-count-weighted differential expression
   #language: r
   #input: dataframe exprDf
   #input: int nGroup1
   #input: dataframe peptideDf
   #input: double fcThreshold = 1.0
   #input: double pThreshold = 0.05
   #output: dataframe result
   ```
   NOTE: `peptideDf` is a single-column dataframe containing integer peptide counts (one per protein row). Using a dataframe instead of a column vector because Datagrok R script inputs support dataframe, int, double, string -- not raw column vectors.

2. **Script body:**
   - Extract expression matrix: `exprMat <- as.matrix(exprDf)`
   - Extract peptide counts: `counts <- peptideDf[, 1]`
   - Build design matrix identical to limma_de.R (group factor with ctrl/treat levels, `model.matrix(~ group)`)
   - Run limma pipeline: `lmFit(exprMat, design)` then `eBayes(fit)`
   - Attempt DEqMS: check `hasDeqms <- suppressWarnings(require(DEqMS, quietly = TRUE))`
   - If DEqMS available:
     - `fit$count <- counts`
     - `fit <- spectraCounteBayes(fit)`
     - `tt <- outputResult(fit, coef = 2)`
     - Build result with DEqMS-specific column names mapped to contract:
       - `log2FC = tt$logFC`
       - `p.value = tt$sca.P.Value`
       - `adj.p.value = tt$sca.adj.pval`
       - `significant = (abs(tt$logFC) >= fcThreshold) & (tt$sca.adj.pval <= pThreshold)`
   - If DEqMS NOT available: fall back to limma `topTable()` (same as limma_de.R):
     - `tt <- topTable(fit, coef = 2, number = Inf, sort.by = "none")`
     - Map: `log2FC = tt$logFC`, `p.value = tt$P.Value`, `adj.p.value = tt$adj.P.Val`
     - Log warning: `warning("DEqMS not available, using limma instead")`
   - Output: `result` dataframe with columns `log2FC`, `p.value`, `adj.p.value`, `significant` (same contract as limma_de.R)
  </action>
