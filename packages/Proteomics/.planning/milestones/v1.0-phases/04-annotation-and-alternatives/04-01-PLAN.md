<objective>
Implement a UniProt info panel that automatically appears in the Datagrok context panel when a user clicks a cell in any column with the `Proteomics-ProteinId` semantic type. The panel fetches protein details from the UniProt REST API and displays protein name, gene, organism, function description, GO terms (grouped by category), and a link to the full UniProt entry.

Purpose: Scientists can quickly inspect any protein in their results without leaving Datagrok -- essential for interpreting DE results and exploring hits.
Output: `uniprot-panel.ts` module + panel registration in `package.ts` and `package.g.ts`.
</objective>

<task type="auto">
  <name>Task 1: Create UniProt panel module with API fetch and widget rendering</name>
  <files>packages/Proteomics/src/panels/uniprot-panel.ts</files>
  <action>
Create `src/panels/uniprot-panel.ts` with the following:

1. **UniProt API fetch function** (`fetchUniProtData`):
   - Endpoint: `https://rest.uniprot.org/uniprotkb/{accession}.json?fields=accession,protein_name,gene_names,organism_name,cc_function,go`
   - Parse the accession from the cell value: handle semicolon-delimited protein groups (e.g., "P12345;Q67890" -> use "P12345"), handle UniProt format prefixes (e.g., "sp|P04637|P53_HUMAN" -> extract "P04637", "tr|Q67890|..." -> extract "Q67890"), and bare accessions.
   - Wrap in try/catch, return null on any failure (network error, 404, parse error).

2. **GO term extraction** (`extractGoTerms`):
   - Parse `uniProtKBCrossReferences` array filtering for `database === 'GO'`.
   - Group by aspect: `F:` -> Molecular Function, `P:` -> Biological Process, `C:` -> Cellular Component.
   - Return `{mf: string[], bp: string[], cc: string[]}`.

3. **Widget rendering** (`renderUniProtWidget`):
   - Build an `HTMLDivElement` container.
   - Show a summary table (using `ui.tableFromMap`) with: Protein name, Gene, Organism, Function (truncated to ~200 chars with ellipsis if longer).
   - Show GO terms grouped by category. Display up to 5 terms per category. Use `ui.divText` with bold category labels ("Molecular Function", "Biological Process", "Cellular Component").
   - Add a clickable link to `https://www.uniprot.org/uniprot/{accession}` at the top using `ui.link` -- prominent placement so the user can easily access the full entry.

4. **Main exported function** (`uniprotPanel`):
   - Accepts `proteinId: string`, returns `DG.Widget`.
   - Extract accession from the raw protein ID value (using the parsing logic above).
   - Return `new DG.Widget(ui.wait(async () => { ... }))` for async loading with built-in spinner.
   - On success: render the widget.
   - On failure (null return from fetch): show `ui.divText('Unable to fetch UniProt data for {accession}')`.
   - On empty/invalid accession: show `ui.divText('No valid UniProt accession found')`.

Use TypeScript interfaces for the UniProt JSON response structure (see RESEARCH.md code examples for the exact JSON paths). Do NOT add any client-side caching -- one request per cell click is fine for this use case.
