  <action>
Create a synthetic demo MaxQuant proteinGroups.txt file at `packages/Proteomics/files/demo/proteinGroups.txt`. This file replaces the `.gitkeep` placeholder.

**Requirements:**
- Tab-delimited text file mimicking MaxQuant proteinGroups.txt output
- Target size: <200KB (small enough for fast loading)
- ~100-150 protein rows total, including:
  - ~90-120 genuine protein entries with realistic UniProt IDs and gene names
  - ~5-8 contaminant entries (some with "+" in "Potential contaminant" column, some with "CON__" prefix in Protein IDs)
  - ~3-5 reverse hit entries (some with "+" in "Reverse" column, some with "REV__" prefix in Protein IDs)
  - ~2-3 only-identified-by-site entries ("+" in "Only identified by site" column)
  - ~2-3 entries with semicolon-delimited Protein IDs (e.g., "P12345;Q67890")
  - ~2-3 entries with semicolon-delimited Gene Names (e.g., "ACTB;ACT")

**Required columns (tab-separated, header row first):**
1. `Protein IDs` -- UniProt accession(s), semicolon-delimited for protein groups
2. `Majority protein IDs` -- primary accession(s)
3. `Gene names` -- gene symbol(s), semicolon-delimited for groups
4. `Protein names` -- human-readable protein names
5. `Potential contaminant` -- "+" for contaminants, empty otherwise
6. `Reverse` -- "+" for reverse hits, empty otherwise
7. `Only identified by site` -- "+" for by-site-only, empty otherwise
8. `LFQ intensity Sample1` through `LFQ intensity Sample6` -- 6 LFQ intensity columns with realistic values:
   - Most values: large positive floats (1e6 to 1e10 range)
   - Some values: 0 (undetected proteins)
   - Some values: empty (missing data)
9. `Sequence coverage [%]` -- percentage value (10-80 range)
10. `Mol. weight [kDa]` -- molecular weight (10-300 range)
11. `Peptides` -- integer count (1-50 range)
12. `Unique peptides` -- integer count (1-40 range)

**Data guidelines:**
- Use real UniProt accession format (e.g., P04406, Q9Y6K1, P68363) but data can be synthetic/approximate
- Use real-looking gene names (e.g., GAPDH, ACTB, TUBB, HSP90AB1, TP53)
- LFQ intensities should span several orders of magnitude to be realistic
- Include a few zero intensity values and a few empty cells to test null/zero handling
- Contaminant entries should have names like "CON__P02768" or just "+" in the Potential contaminant column
- Keep the file well-formed: every row has the same number of tab-separated fields as the header

**Do NOT:**
- Download a file from the internet -- create it synthetically
- Include more than ~150 rows -- keep it small
- Include columns beyond the ~14 listed above -- keep it minimal
