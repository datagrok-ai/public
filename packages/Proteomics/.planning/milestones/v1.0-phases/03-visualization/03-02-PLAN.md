4. Select top N proteins: sort by adj.p-value ascending, take the first topN rows that have non-null adj.p-value. Collect their row indices into a Set.
5. **Row filtering via BitSet:** Use the DataFrame's filter (BitSet) to show only the top N protein rows in the heatmap Grid. Apply this to the Grid's row source:
   ```typescript
   // Create the grid FIRST on the original df
   const grid = df.plot.grid();
   // Set the DataFrame filter to show only topN rows
   // Use df.filter to control which rows are visible
   const filter = DG.BitSet.create(df.rowCount);
   for (const idx of topNIndices)
     filter.set(idx, true);
   grid.dataFrame.filter.copyFrom(filter);
   ```
   Note: If modifying `df.filter` directly would affect other viewers (volcano), instead use `grid.setOptions({rowSource: 'Filtered'})` and manipulate a temporary selection, OR create the grid and use `grid.dataFrame.rows.match()` / `grid.sort()` to control visible rows. The key constraint is: the Grid's `dataFrame` property MUST reference `df` (the same object), not a copy. Test which filtering approach works without side-effecting the volcano plot's row visibility. If `df.filter` is shared across viewers, use `df.rows.select()` on the topN indices and set the grid's row source to 'Selected', then restore selection afterward. Document the approach chosen.
6. **Grid column visibility:** Hide all non-intensity columns except the label column. Show only intensity columns grouped by experimental condition:
   ```typescript
   // Hide all columns first
   for (let i = 0; i < grid.columns.length; i++) {
     const gc = grid.columns.byIndex(i);
     if (gc) gc.visible = false;
   }
   // Show label column and intensity columns in group order
   const labelGc = grid.columns.byName(labelCol);
   if (labelGc) labelGc.visible = true;
   // Show group1 columns first (control), then group2 (treatment)
   for (const colName of [...groups.group1.columns, ...groups.group2.columns]) {
     const gc = grid.columns.byName(colName);
     if (gc) gc.visible = true;
   }
   ```
   This ensures Grid heatmap mode only renders intensity columns as colored cells, without needing a separate DataFrame.
7. **Z-score color normalization:** Instead of transforming the data values (which would corrupt the original DataFrame), apply z-score normalization visually via per-column color coding settings. Set each intensity column's color coding to use a linear scheme based on the row's z-score range. Alternatively, if Grid heatmap mode does not support per-column color coding configuration, add temporary z-score columns (named e.g. `_zscore_{colName}`) to the DataFrame, show those instead of the raw intensity columns, and hide the raw ones. If using temporary columns, document that they are heatmap-internal and should be cleaned up when the heatmap is closed.

   - If the Dendrogram package is not installed or the call fails, fall back to sorting rows by adj.p-value (ascending). Log a warning: `console.warn('Dendrogram package not available; falling back to significance-based row ordering')`.
   - This graceful degradation ensures the heatmap works without the Dendrogram dependency while honoring the user's clustering preference when possible.

```typescript
//top-menu: Proteomics | Visualize | Volcano Plot...
@grok.decorators.func()
static async showVolcanoPlot(): Promise<void> {
  const tv = grok.shell.tv;
  const df = tv?.dataFrame;
  if (!df) { grok.shell.warning('No table open'); return; }
  if (df.getTag('proteomics.de_complete') !== 'true') {
    grok.shell.warning('Run Differential Expression first (Proteomics | Analyze | Differential Expression)');
    return;
  }
  const sp = createVolcanoPlot(df);
  tv.addViewer(sp);
}

//tags: menuItem
//top-menu: Proteomics | Visualize | Heatmap...
@grok.decorators.func()
static async showHeatmap(): Promise<void> {
  const tv = grok.shell.tv;
  const df = tv?.dataFrame;
  if (!df) { grok.shell.warning('No table open'); return; }
  if (df.getTag('proteomics.de_complete') !== 'true') {
    grok.shell.warning('Run Differential Expression first (Proteomics | Analyze | Differential Expression)');

//top-menu: Proteomics | Visualize | PCA...
@grok.decorators.func()
static async showPcaPlot(): Promise<void> {
  const tv = grok.shell.tv;
  const df = tv?.dataFrame;
  if (!df) { grok.shell.warning('No table open'); return; }
  // PCA does NOT require DE -- it works after import/normalization
  const groups = getGroups(df);
  if (!groups) {
    grok.shell.warning('Annotate experimental groups first (Proteomics | Annotate Experiment)');
    return;
  }
  const allCols = [...groups.group1.columns, ...groups.group2.columns];
  const sp = createPcaPlot(df, allCols, groups);
  // PCA creates its own sample-level DataFrame -- open in separate table view
  grok.shell.addTableView(sp.dataFrame);
}
```

```typescript
static async showAllVisualizations(): Promise<void> {
  const tv = grok.shell.tv;
  const df = tv?.dataFrame;
  if (!df) { grok.shell.warning('No table open'); return; }
  if (df.getTag('proteomics.de_complete') !== 'true') {

  const groups = getGroups(df);
  // Volcano -- docked in current table view (protein-level)
  const volcano = createVolcanoPlot(df);
  tv.addViewer(volcano);
  // Heatmap -- docked in current table view (protein-level)
  const heatmap = await createExpressionHeatmap(df);
  tv.addViewer(heatmap);
  // PCA (if groups available) -- opens in SEPARATE table view (sample-level, different row count)
  if (groups) {
    const allCols = [...groups.group1.columns, ...groups.group2.columns];
    const pcaPlot = createPcaPlot(df, allCols, groups);
    grok.shell.addTableView(pcaPlot.dataFrame);
  }
}
```
