**Create `packages/Proteomics/src/analysis/pca.ts`** with:

**`computePCA(df: DG.DataFrame, intensityCols: string[], groupAssignment: GroupAssignment, nComponents: number = 2): {pcaDf: DG.DataFrame; varianceExplained: number[]}`**

Algorithm for client-side sample-level PCA on typical proteomics data (rows = proteins, cols = samples, where nSamples < 50):

1. Build matrix: nSamples x nProteins (TRANSPOSED -- each sample is a row of protein intensities across all proteins). For each sample column in intensityCols, read all protein rows to form one row in the matrix. If null/NaN, impute with the protein mean (column mean in original DataFrame; row mean in transposed matrix). This imputation is for PCA only -- does not modify the original DataFrame.
2. Center each column (protein): subtract the column mean from every value.
3. Compute the covariance matrix S (nSamples x nSamples): S = X * X^T / (nProteins - 1). This is a small matrix (typically < 50x50).
4. Eigendecompose S using Jacobi rotation method:
   - Iterate: find largest off-diagonal element, apply rotation to zero it.
   - Convergence when max off-diagonal < 1e-10 or after 100 iterations.
   - Returns eigenvalues and eigenvectors sorted descending by eigenvalue.
5. PC scores are directly the eigenvectors scaled by sqrt(eigenvalue) for each sample.
6. Compute variance explained: eigenvalue[i] / sum(eigenvalues) * 100.
7. Create a NEW sample-level DataFrame (NOT added to the original protein DataFrame -- different row count):
   - Column `SampleName` (string): the intensity column names from intensityCols.
   - Column `PC1 (45.2%)` (float): PC1 score for each sample (variance explained in name).
   - Column `PC2 (23.1%)` (float): PC2 score for each sample.
   - Column `Group` (string): map each sample to its group name using groupAssignment (check if sample is in group1.columns or group2.columns).
8. Return `{pcaDf, varianceExplained: [45.2, 23.1, ...]}`.

Helper functions (not exported, internal):
- `jacobi(matrix: number[][], n: number): {eigenvalues: number[]; eigenvectors: number[][]}` -- Jacobi rotation eigendecomposition.
- `transpose(matrix: number[][], rows: number, cols: number): number[][]` -- matrix transpose.
- `matmul(A: number[][], B: number[][], m: number, n: number, p: number): number[][]` -- matrix multiply.

**Create `packages/Proteomics/src/viewers/pca-plot.ts`** with:

**`createPcaPlot(df: DG.DataFrame, intensityCols: string[], groupAssignment: GroupAssignment): DG.ScatterPlotViewer`**

1. Call `computePCA(df, intensityCols, groupAssignment)` which returns `{pcaDf, varianceExplained}`.
2. Find the PC column names on pcaDf (they include variance explained in the name, e.g., `PC1 (45.2%)`).
3. Create scatter plot on the NEW sample-level DataFrame: `pcaDf.plot.scatter({x: pc1ColName, y: pc2ColName, color: 'Group'})`.
4. Label all sample points: set `sp.props.labelColumnNames = ['SampleName']` and `sp.props.displayLabels = 'Always'` (typical proteomics datasets have <50 samples so labeling all is appropriate).
5. **95% confidence ellipses:** Compute 95% confidence ellipse parameters per group using the PC1/PC2 scores. For each group, compute the mean (cx, cy) and 2x2 covariance matrix of PC1/PC2 scores, then derive ellipse semi-axes and rotation angle from eigendecomposition. Scale by chi-squared critical value (5.991 for 2 DOF at 95%). Approximate each ellipse as a polygon (e.g., 64 points) and add via `sp.meta.formulaLines.addLine()` as a band or via `AreaAnnotationRegion` with the `area` field containing the polygon points. Use the Datagrok `AreaAnnotationRegion` interface: `{type: 'area', area: [[x1,y1], [x2,y2], ...], fillColor: groupColor, opacity: 0.15, outlineColor: groupColor, outlineWidth: 1}`. If the annotation region API does not render correctly at runtime, document as a known limitation and note that ellipses can be added in v2 with a custom overlay.
6. Return the ScatterPlotViewer.

**Important architectural note:** PCA creates a NEW sample-level DataFrame (nSamples rows, not nProteins rows). This means:
- PCA does NOT share the same DataFrame as volcano/heatmap.
- Linked selection between volcano/heatmap (protein-level) and PCA (sample-level) is NOT applicable -- they represent different entities (proteins vs samples). This is architecturally correct and standard in proteomics tools.
- PCA opens in a separate table view.
