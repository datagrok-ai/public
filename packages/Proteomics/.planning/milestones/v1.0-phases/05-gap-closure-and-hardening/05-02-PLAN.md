Three fixes in detectors.js:

1. **isLog2Intensity (line 75):** Add `log2 ` (space, case-insensitive) as alternative prefix. MaxQuant also produces columns like "Log2 LFQ intensity Sample1" (space instead of paren). The current `.toLowerCase()` already handles case. Change the check to:
   ```javascript
   const isLog2Intensity = (name.startsWith('log2(') || name.startsWith('log2 ')) &&
     (name.includes('intensity') || name.includes('ibaq'));
   ```

2. **isRawIntensity (line 73-74):** Remove the `&& col.min >= 0` guard. When a column has all null/NaN values, `col.min` returns NaN and `NaN >= 0` evaluates to `false`, causing detection to fail. The name prefix is specific enough (intensity, lfq intensity, reporter intensity, ibaq) that the min guard is unnecessary.
   Change to:
   ```javascript
   const isRawIntensity = name.startsWith('intensity') || name.startsWith('lfq intensity') ||
     name.startsWith('reporter intensity') || name.startsWith('ibaq');
   ```

3. **UniProt regex (line 13):** The regex alternation `^[OPQ][0-9][A-Z0-9]{3}[0-9]|[A-NR-Z][0-9]([A-Z][A-Z0-9]{2}[0-9]){1,2}$` is missing grouping -- the `^` only anchors the first alternative, and `$` only anchors the second. Wrap in a non-capturing group:
   ```javascript
   const uniprotPattern = /^(?:[OPQ][0-9][A-Z0-9]{3}[0-9]|[A-NR-Z][0-9](?:[A-Z][A-Z0-9]{2}[0-9]){1,2})$/;
   ```
