const isRawIntensity = (name.startsWith('intensity') || name.startsWith('lfq intensity') ||

**Step 2: Replace the clustering block**

Replace the entire clustering try/catch block (section 7 in the current code) with:

```typescript
let clustered = false;
try {
  const treeHelper = await getTreeHelper();
  const dendrogramService = await getDendrogramService();

  // Use TreeHelper to compute distance matrix on z-score columns of the cloned DataFrame
  const distMatrix = await treeHelper.calcDistanceMatrix(heatmapDf, zScoreColNames, 'euclidean');
  if (distMatrix) {
    // Cluster and get tree root
    const treeRoot = await treeHelper.hierarchicalClusteringByDistance(distMatrix, 'average');

    // Reorder grid rows by leaf order
    treeHelper.setGridOrder(treeRoot, grid);

    // Inject dendrogram tree visual alongside the grid
    dendrogramService.injectTreeForGrid(grid, treeRoot, undefined, 150);
    clustered = true;
  }
} catch (_e) {
  console.warn('Dendrogram package not available; falling back to significance-based row ordering');
}
```

Key points:
- `calcDistanceMatrix` works on the cloned `heatmapDf` with z-score column names — no hand-rolled distance matrix needed
- `hierarchicalClusteringByDistance` replaces the old `dendrogramService.getTree()` call (which was a non-standard API usage)
- `setGridOrder` reorders the grid rows by the tree's leaf order — replaces the manual sort column hack
- `injectTreeForGrid` renders the tree visual as a `GridNeighbor` panel on the left side of the grid (150px width)
- The `leafColName` parameter is `undefined` because the tree uses row indices, not a named column
- If the Dendrogram package is not installed, the catch block falls through to significance-based sorting (same as before)
