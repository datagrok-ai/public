# Roadmap: Proteomics Package

## Overview

This roadmap delivers a Datagrok extension package that takes scientists from MaxQuant proteinGroups.txt output through differential expression analysis with interactive visualizations. The four phases follow the natural data flow: get data in (import), analyze it (normalize, impute, DE), see results (volcano, heatmap, PCA), and enrich the experience (UniProt annotation, DEqMS alternative). Each phase delivers a coherent, testable capability that builds on the previous one.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: Data Import and Foundation** - MaxQuant import with auto-detection, filtering, semantic types, and demo dataset
- [ ] **Phase 2: Analysis Pipeline** - Experiment annotation, normalization, imputation, and differential expression (Welch's t-test + BH FDR)
- [ ] **Phase 3: Visualization** - Volcano plot, heatmap with clustering, and PCA colored by group
- [ ] **Phase 4: Annotation and Alternatives** - UniProt info panel and DEqMS as alternative DE method
- [ ] **Phase 5: Gap Closure & Hardening** - Fix integration bugs (R script name, detector regex, heatmap filter) and render dendrogram on heatmap

## Phase Details

### Phase 1: Data Import and Foundation
**Goal**: Scientists can import a MaxQuant proteinGroups.txt file and get a clean, properly typed DataFrame with contaminants filtered, intensities log2-transformed, and all columns classified by semantic type
**Depends on**: Nothing (first phase)
**Requirements**: IMPORT-01, IMPORT-02, IMPORT-03, IMPORT-04, IMPORT-05
**Success Criteria** (what must be TRUE):
  1. User can open a MaxQuant proteinGroups.txt file and see a filtered DataFrame with contaminant and reverse hits removed
  2. Intensity columns are automatically detected (LFQ, iBAQ, or Reporter) and log2-transformed on import
  3. Protein ID, gene name, and intensity columns have correct semantic types visible in the column header tooltips
  4. User can open the bundled demo dataset from the package and see properly typed proteomics data ready for analysis
**Plans**: 2 plans

Plans:
- [ ] 01-01-PLAN.md -- MaxQuant parser implementation + import menu wiring
- [ ] 01-02-PLAN.md -- Demo dataset, demo loader, and parser tests

### Phase 2: Analysis Pipeline
**Goal**: Scientists can take imported proteomics data through the complete analysis pipeline -- assign experimental groups, normalize, impute missing values, and run differential expression -- producing log2FC and p-value columns ready for visualization
**Depends on**: Phase 1
**Requirements**: SETUP-01, SETUP-02, ANLY-01, ANLY-02, ANLY-03, ANLY-05
**Success Criteria** (what must be TRUE):
  1. User can open an annotation dialog, assign sample columns to two experimental groups, and see group assignments persisted as DataFrame metadata
  2. User can apply median centering normalization to intensity columns and observe the median shift to zero across samples
  3. User can impute missing values using MinProb and observe that NaN cells in intensity columns are filled with low-abundance values
  4. User can run differential expression (Welch's t-test + BH FDR) between two groups and see new log2FC, p-value, and adjusted p-value columns added to the DataFrame with correct semantic types
**Plans**: 3 plans

Plans:
- [ ] 02-01-PLAN.md -- Experiment annotation, normalization, and imputation with dialogs
- [ ] 02-02-PLAN.md -- Differential expression (Welch's t-test + BH FDR) with significance column
- [ ] 02-03-PLAN.md -- Analysis pipeline unit tests

### Phase 3: Visualization
**Goal**: Scientists can visualize differential expression results with interactive, linked proteomics viewers that follow community conventions
**Depends on**: Phase 2
**Requirements**: VIZ-01, VIZ-02, VIZ-03
**Success Criteria** (what must be TRUE):
  1. User can open a volcano plot showing log2FC vs -log10(adjusted p-value) with horizontal and vertical threshold lines marking significance cutoffs
  2. User can open a heatmap of top differentially expressed proteins with hierarchical clustering and sample grouping
  3. User can open a PCA plot colored by experimental group that reveals sample clustering and potential outliers
  4. Selecting a protein in one viewer highlights the same protein across all open viewers (linked selection via shared DataFrame)
**Plans**: 2 plans

Plans:
- [ ] 03-01-PLAN.md -- Volcano plot factory and PCA computation/plot
- [ ] 03-02-PLAN.md -- Expression heatmap and menu wiring

### Phase 4: Annotation and Alternatives
**Goal**: Scientists can inspect individual proteins via UniProt and have an alternative DE method (DEqMS) that accounts for peptide count
**Depends on**: Phase 1 (UniProt panel), Phase 2 (DEqMS)
**Requirements**: ANNOT-01, ANNOT-02, ANLY-04
**Success Criteria** (what must be TRUE):
  1. User can click a protein ID in the grid and see UniProt details (protein name, function description, GO terms) appear in the context panel
  2. UniProt info panel triggers automatically for any column with the Proteomics:ProteinId semantic type without manual configuration
  3. User can run DEqMS differential expression as an alternative to limma and see results with log2FC, p-value, and adjusted p-value columns
**Plans**: 2 plans

Plans:
- [ ] 04-01-PLAN.md -- UniProt info panel with semantic type auto-trigger
- [ ] 04-02-PLAN.md -- DEqMS differential expression R script and dialog integration

### Phase 5: Gap Closure & Hardening
**Goal**: Close remaining audit integration risks, render the dendrogram tree visual alongside the heatmap, and fix UAT-reported issues with detector regex and performance
**Depends on**: Phases 1-4 (all existing work)
**Requirements**: VIZ-02, IMPORT-02, IMPORT-04, ANLY-04
**Gap Closure:** Closes integration risks from v1.0 audit + heatmap dendrogram rendering + UAT gap closure
**Success Criteria** (what must be TRUE):
  1. `grok.functions.call('Proteomics:limmaDE')` matches the actual registered function name — DEqMS path no longer silently falls back to t-test
  2. `detectors.js::detectIntensity` recognizes `log2(...)` and `Log2 ...` column names — semTypes survive project save/reload
  3. Heatmap top-50 filter does not leak to volcano plot when both viewers are open simultaneously
  4. Heatmap displays a dendrogram tree visual alongside clustered rows (not just sorted order)
  5. Raw intensity detection works on columns with all-null values (no col.min guard)
  6. Heatmap creation shows progress indicator to user
  7. R limma script fails fast when limma unavailable (no slow for-loop fallback)
**Plans**: 3 plans
