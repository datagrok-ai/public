~/Downloads/ck/CKomics_tool2.py lines 4685-4736: apply_smart_pathway_filtering — the function being ported.
Critical contract elements (per 14-RESEARCH.md §"Pattern 7" + Assumption A4):

1. GENERIC_PARENT_TERMS (CK-omics literal list, case-insensitive substring match):
   ['localization', 'cellular component organization', 'transport',
    'cellular process', 'biological process', 'metabolic process']

2. SPECIFIC_CHILD_TERMS (CK-omics literal list, case-insensitive substring match):
   ['actin', 'vesicle', 'endocytosis', 'cytoskeleton']

3. Algorithm (line 4685-4736):
   a) Partition input into GO:BP rows and non-GO:BP rows (other_data).
   b) Sort GO:BP by p_value ASCENDING.
   c) For each GO:BP row in sorted order:
      - if name (lowercased) contains any GENERIC_PARENT term, AND any already-kept GO:BP row contains a SPECIFIC_CHILD term in its name → DROP this row (count as droppedParents).
      - otherwise → KEEP.
      - if kept count reaches maxPerSource (default 15) → break out of GO:BP loop.
   d) For non-GO:BP rows: sort by p_value ASCENDING, take top `maxPerSource` (combined cap, NOT per-source — CK-omics line 4731 explicitly does `.head(maxPerSource)` on the combined `other_data`; Assumption A4 in RESEARCH).
   e) Concatenate filteredGoBp + otherCapped, sort by p_value ASCENDING, return.

4. Stats returned: {total, kept, droppedParents, cappedAtN: maxPerSource}.
