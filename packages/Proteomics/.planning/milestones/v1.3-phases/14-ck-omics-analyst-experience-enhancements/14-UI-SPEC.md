# Phase 14 — UI Design Contract

> Visual and interaction contract for CK-omics analyst-experience enhancements. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 14 layers UX onto existing Datagrok-native viewers (volcano scatter, UniProt context panel, enrichment table) and adds one new viewer (Group-Mean Correlation scatter). All design tokens are dictated by the host Datagrok platform; this contract codifies the package-specific overrides locked in `14-CONTEXT.md` (D-01..D-14).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Datagrok TS package — no component framework) |
| Preset | not applicable |
| Component library | Datagrok-native primitives: `ui.div`, `ui.divH`, `ui.divV`, `ui.divText`, `ui.label`, `ui.button`, `ui.bigButton`, `ui.link`, `ui.icons`, `ui.tooltip.bind`, `ui.tableFromMap`, `ui.wait`, `ui.dialog`, `ui.input.*`, `DG.Viewer`, `DG.ScatterPlotViewer`, `DG.Widget`, `DG.TaskBarProgressIndicator` |
| Icon library | `ui.icons.*` (Datagrok built-in Font Awesome wrappers) |
| Font | Inherited from Datagrok platform (`datagrok-api/css/styles.css`) — no override |

**Rationale:** Datagrok packages render inside the host shell. Adding shadcn or any external CSS framework conflicts with platform theming, breaks dark-mode switching, and violates the established package pattern (no Proteomics-package `css/` folder exists in `packages/Proteomics/`). All viewers in this package use `DG.Viewer.*` factories with inline `style.*` overrides only where strictly necessary (see existing `src/panels/uniprot-panel.ts:118-122`).

## Spacing Scale

Datagrok uses an implicit 4-px scale. Declared values for Phase-14 additions (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Marker-glyph gap, inline-text spacing inside bar-chart whiskers |
| sm | 8px | UniProt-panel section gap (matches existing `uniprot-panel.ts:120,156` `marginBottom: 8px` / `marginTop: 8px`) |
| md | 16px | Counter-overlay internal padding, dialog-input row gap |
| lg | 24px | Section padding inside the per-group bar chart container |

**Exceptions:**
- Counter overlay anchor: `bottom: 8px; right: 8px` (D-06 floating, not docked) — 8px matches the existing platform viewer-frame inset.
- Volcano default-label spacing: governed by the native `DG.ScatterPlotViewer` `labelColumnNames` engine; no override.
- Per-group bar-chart bar width / gap: `barWidth: 24px`, `barGap: 8px` inside `uniprot-panel.ts` (D-11 compact bars).

**Forbidden:** any non-multiple-of-4 value, any `px` value above 64 in new code (must use platform layout instead).

## Typography

Datagrok platform sets the base. Phase-14 declarations are RELATIVE so they survive theme switches:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (panel text, counter values, axis labels) | inherit (platform default) | 400 | 1.4 (platform) |
| Label (counter category names, GO-term headers, per-group bar labels) | 0.85em (relative) | 600 | 1.3 |
| Heading (panel section headers — "GO Terms", "Per-Group Quantities") | inherit | 700 | 1.3 |
| Display (volcano title — synthesized "Volcano Plot: <Num> vs <Denom>") | inherit (use `sp.setOptions({title})` — no font override) | platform | platform |

**Rules:**
- Always use `em` / `inherit` for sizes — never hardcode `px`. This mirrors the existing pattern in `src/panels/uniprot-panel.ts:170` (`fontSize: '0.85em'`).
- Two weights only: 400 (normal) and 600/700 (bold). Use `font-weight: bold` not numeric values to follow `uniprot-panel.ts:119,155,162,168` convention.
- The provenance markers `*` (grouped) and `†` (predicted) are plain text characters in the `Display Name` string — NOT icons, NOT font-styled. They must render correctly in the volcano label layer, the grid cells, the enrichment table, and tooltips with zero CSS.

## Color

Datagrok platform owns the global palette. Phase-14 introduces ONE locked semantic palette (D-04) on top:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | platform surface (theme-driven) | DataFrame grids, viewer backgrounds, panel surfaces |
| Secondary (30%) | platform card/sidebar (theme-driven) | UniProt panel container, counter-overlay background (semi-transparent), dialog backgrounds |
| Accent — Numerator-enriched | `#FF00FF` (magenta — ARGB `0xFFFF00FF`) | Volcano points classified `"Enriched in <group1>"`, group1 bar in UniProt per-group chart, group1 cluster in Group-Mean Correlation scatter |
| Accent — Denominator-enriched | `#00FFFF` (cyan — ARGB `0xFF00FFFF`) | Volcano points classified `"Enriched in <group2>"`, group2 bar in UniProt per-group chart, group2 cluster in Group-Mean Correlation scatter |
| Neutral — Not significant | `#AAAAAA` (gray — ARGB `0xFFAAAAAA`) | Volcano NS points, NS cluster in Group-Mean Correlation scatter (matches existing `volcano.ts:84` `'not significant': 0xFFAAAAAA`) |
| Subcellular Location palette | LOCKED 11+1 hex map | Volcano color-mode = Location ONLY (Phase 13 D-04 carry-forward — `src/analysis/subcellular-location.ts:51-64`) |
| Destructive | platform-default red (`grok.shell.warning` toast) | Duplicate-description warning (D-10), Ensembl-resolution failure warning (R1 graceful degrade) |

**Accent reserved for:**
- Volcano point coloring when `colorDim === 'significance'` (the default mode).
- Per-group bar fill in the UniProt panel (D-11) — bar 1 = magenta, bar 2 = cyan.
- Group-Mean Correlation scatter point coloring by significance class (D-12).

**Accent is NOT used for:**
- Dialog accent / button highlight (use platform default `ui.bigButton`).
- Counter overlay text (use platform text color).
- Volcano direction-line color (must stay `#888888` per `volcano.ts:184` to keep threshold lines visually distinct from data).
- Subcellular-location mode coloring (uses the 11-color palette, NOT magenta/cyan/gray).
- Provenance markers (plain text, no color override).

**Color lock is non-negotiable (CONTEXT.md "Specifics"):**
- Magenta = group1 (numerator) — same hex everywhere.
- Cyan = group2 (denominator) — same hex everywhere.
- Gray = NS — same hex everywhere.
- Defined ONCE in a constant (e.g. `DIRECTION_COLORS` in `src/viewers/volcano.ts`) and imported by every consumer. Inline hex literals in any other file are a design-contract violation.

**Existing volcano color map (`volcano.ts:84`) must be MIGRATED:**
- Current: `{'up': 0xFFCC0000, 'down': 0xFF0000CC, 'not significant': 0xFFAAAAAA}` (red/blue/gray)
- Phase 14: `{'Enriched in <group1>': 0xFFFF00FF, 'Enriched in <group2>': 0xFF00FFFF, 'Not significant': 0xFFAAAAAA}` (magenta/cyan/gray, with group names resolved from `proteomics.groups` tag — D-04)
- Category-string change cascades through `ensureDirectionColumn`, the enrichment cross-link logic in `enrichment-viewers.ts:wireEnrichmentToVolcano`, and any test fixture asserting on the literal `'up'` / `'down'` strings.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Volcano title (synthesized from `proteomics.de_method` + group names) | `"Volcano Plot: {group1} vs {group2}"` (e.g. `"Volcano Plot: DMD vs WT"`) — set via `sp.setOptions({title})`; no fallback string when groups are unset, use `"Volcano Plot"` |
| Volcano X-axis label (live-rewrite on metric toggle) | `"Log2 Fold Change"` |
| Volcano Y-axis label — adj.p-value mode | `"-Log10(Q-value)"` |
| Volcano Y-axis label — p-value mode | `"-Log10(p-value)"` |
| Volcano legend categories (read group names live from `proteomics.groups` JSON via `getGroups(df)`) | `"Enriched in {group1}"`, `"Enriched in {group2}"`, `"Not significant"` |
| Counter overlay heading | `"Visible Proteins"` |
| Counter overlay rows (significance mode) | `"Total: {n}"`, `"Enriched in {group1}: {n}"`, `"Enriched in {group2}: {n}"`, `"Not significant: {n}"` (one row per category) |
| Counter overlay rows (location mode) | `"Total: {n}"` then one row per location category present: `"{Location}: {n}"` (use the LOCKED 11-category names from `subcellular-location.ts:LOCATION_KEYWORDS` + `"Unknown"`) |
| UniProt panel — per-group section header | `"Per-Group Quantities"` (bold, matches existing `"GO Terms"` header in `uniprot-panel.ts:153`) |
| UniProt panel — bar caption format | `"{group} (n={N})"` below each bar; `"mean={x.xx}  SD={x.xx}"` in small text below the caption |
| UniProt panel — missing-quantity message | `"No per-group quantities available for this protein"` (when `proteomics.groups` tag is absent or the protein has all-NaN values in both groups) |
| Group-Mean Correlation title | `"Group-Mean Correlation — r={r:.2f} (Pearson), ρ={rho:.2f} (Spearman)"` (e.g. `"Group-Mean Correlation — r=0.93 (Pearson), ρ=0.91 (Spearman)"`) |
| Group-Mean Correlation X-axis | `"{group1} mean (log2 intensity)"` |
| Group-Mean Correlation Y-axis | `"{group2} mean (log2 intensity)"` |
| Group-Mean Correlation legend | reuses volcano legend categories (`"Enriched in {group1}"`, `"Enriched in {group2}"`, `"Not significant"`) |
| Enrichment dialog toggle (R5) | `"Apply smart pathway filter"` (checkbox, default-checked); tooltip on hover: `"Drops generic GO parent terms when more specific child terms are present; caps each source at top-N by FDR. Recommended for cleaner results; uncheck for raw g:Profiler output."` |
| Enrichment-table footer banner (when smart filter dropped rows) | `"Smart pathway filter active: showing {kept} of {total} terms (dropped {parents} generic parents, capped at {capN} per source). Re-run with the filter off to see all."` (rendered as a `ui.divText` strip docked above the enrichment grid; use platform info color, not destructive) |
| Display Name marker semantics (tooltip on hover over a marked label) | `*` → `"Grouped: multiple source IDs resolve to this name"`; `†` → `"Predicted / reclassified gene name (Ensembl description fallback)"`; both: `"Original ID: {Source ID}"` appended on a new line |
| Empty state — volcano counter overlay with zero filtered rows | `"Visible Proteins: Total: 0"` (still shows; do not hide the overlay — the zero count IS information) |
| Empty state — UniProt panel with no protein selected | `"Click a point on the volcano plot to see protein details"` (the existing panel already shows `"No valid UniProt accession found"` when the click yields no accession — keep that string unchanged) |
| Error state — Ensembl resolution failure (R1 graceful degrade) | `grok.shell.warning("Ensembl gene-label resolution unavailable. Showing raw IDs ({n} unresolved). Re-import to retry.")` — single toast, do not block the parser |
| Error state — duplicate-description warning (D-10) | `grok.shell.warning("{n} duplicate gene names disambiguated with source IDs.")` — single toast, fires once per import |
| Error state — Color → Location fetch failure (G3 carry-forward) | `grok.shell.warning("Subcellular-location fetch failed. Falling back to significance coloring.")` |
| Destructive confirmation | None — Phase 14 has NO destructive actions. All transforms are additive (new columns, re-color, re-label). Re-running smart-pathway-filter or Color toggle replaces in place; no data loss path exists |

**Copy rules:**
- Never use `"data"` as a noun phrase — use `"proteins"`, `"terms"`, `"points"`, `"rows"` specific to the surface.
- Group names ALWAYS resolved live from `getGroups(df)` — never hardcode `"Control"` / `"Treatment"`.
- Counter values ALWAYS update from the FILTERED set (`df.filter.trueCount` for total, then split by category over the same filter mask).
- The smart-pathway-filter banner is mandatory whenever `kept < total` — silent transforms are explicitly forbidden by the project_context constraint.

## Interaction Contracts

### Volcano: live counter overlay (R2, D-06)

- **Placement:** floating, anchored `bottom: 8px; right: 8px` over the volcano viewer canvas. NOT docked. NOT inside the legend.
- **Container:** `ui.divV([...])` with `position: absolute`, `bottom: 8px`, `right: 8px`, `padding: 8px 16px`, `background: rgba(255,255,255,0.85)` (light theme) — inverted via `var(--grey-1)` if available; otherwise use a single semi-transparent neutral that reads on both themes.
- **Z-index:** above the scatter canvas, below platform dialogs/toolbars (do not set z-index above the dialog layer).
- **Update triggers (ALL must wire):**
  - `df.onFilterChanged.subscribe(...)` — when the user toggles a Filters-viewer category or types into the search box.
  - `df.selection.onChanged.subscribe(...)` — selection-driven highlight from the search match (D-05) does not change counts, but the contract still requires recompute-on-selection for parity with CK-omics behavior.
  - `sp.onPropertyChanged.subscribe(...)` — fires on metric toggle (Q/P) and color-mode toggle (significance/location).
- **Recompute logic:** iterate `df.filter` bits; count true bits per category column (`direction` or `Subcellular Location`); rerender DOM in place (do not destroy + recreate — avoids flicker).
- **Subscription cleanup:** mirror the existing `enrichment-viewers.ts:9` `activeSubscriptions` pattern — store subscriptions and dispose on viewer detach.
- **Empty filter (no rows visible):** still render the overlay with `Total: 0` and one zero-count row per category. Do NOT hide.

### Volcano: default top-N labels (R2 / D-03)

- **Trigger:** initial render, and any of: metric toggle, color-mode toggle, filter change.
- **N:** 15 (locked default; expose as `topNLabels` option on `createVolcanoPlot` for tests).
- **Selection:** top 15 rows by ASCENDING active significance metric over the filtered set, then by ABSOLUTE log2FC desc as tiebreak.
- **Mechanism:** set `sp.props.labelColumnNames = [DISPLAY_NAME_COL]` and use `sp.props.showLabelsFor = 'Selected'` plus drive selection via the top-N indices — OR if the platform supports per-row label-visibility set it directly. Planner picks the exact mechanism.
- **Hover labels:** preserve existing `showLabelsFor = 'MouseOverRow'` behavior in addition to the top-N defaults (`volcano.ts:212`).
- **Label content:** must be `Display Name` (with `*` / `†` markers inline) — NEVER `Source ID`.

### Volcano: protein search via unified Filters viewer (D-05, G4)

- **Single Filters viewer** docked left of the volcano. `columnNames` MUST be exactly: `['Comparison (group1/group2)', 'Display Name', 'Source ID']` — explicit allowlist, `Flags` excluded.
- **Mechanism for G4 enforcement:** if `DG.Viewer.filters(df, {columnNames})` does not honor the allowlist (Phase 13 observation), call `viewer.setOptions({columnNames})` post-creation OR use `DG.Viewer.fromType('Filters', df, {...})` with the platform's column-exclusion option. Planner verifies and picks.
- **Search-match wiring:** typing in the string-column search box triggers Datagrok's built-in filter behavior. To convert filter-matches to highlight-instead-of-hide, planner subscribes to the relevant filter event and calls `df.selection.set(idx, true)` for matched rows while LEAVING `df.filter` untouched. NS points stay visible in the cloud — matched points render in selection color in place.
- **Reset affordance:** "Clear search" button (platform default Filters viewer reset).

### Volcano: metric toggle (carry-forward from Phase 13 D-05/D-06)

- Live property toggle (no dialog re-open required). Already wired via `recomputeVolcano` (`volcano.ts:231-250`). Phase 14 adds: axis-label live-rewrite (X label is invariant, Y label changes per metric — see Copywriting).

### Volcano: color-mode toggle (carry-forward + G3 progress)

- Significance ↔ Location toggle. Already wired via `recomputeVolcano(..., colorDim, ...)`.
- **G3 fix:** wrap the Location path in `DG.TaskBarProgressIndicator` (planner picks between `DG.TaskBarProgressIndicator` and `ui.setUpdateIndicator(viewerHost, true)`). The indicator MUST appear immediately on click, persist until `ensureLocationColumn` resolves, and dismiss in a `finally` block so a failure also clears it.

### Volcano Options dialog: state preload (G2)

- Re-opening the dialog MUST show the values currently applied to the viewer.
- Source-of-truth precedence: (1) `sp.getOptions()` for viewer-bound props (color mode, metric, top-N), (2) `df.tags` for pipeline state (current contrast, group names), (3) last-input cache as absolute fallback. Planner picks and codifies.
- Every input in the dialog reads from this precedence chain on open; `OK` writes back via `sp.setOptions(...)` + `recomputeVolcano(...)`.

### UniProt panel: per-group quantity chart (R3 / D-11)

- **Placement:** appended to the existing `renderUniProtWidget(...)` return container (`uniprot-panel.ts:109-176`), AFTER the GO Terms section, BEFORE the closing return.
- **Container:** `ui.divV([sectionHeader, barChart])` with `marginTop: 8px`.
- **Section header:** `ui.divText("Per-Group Quantities")` styled to match the GO Terms header (`fontWeight: bold; marginTop: 8px; marginBottom: 4px`).
- **Bar chart:** lightweight canvas or SVG (no new viewer). One bar per group from `getGroups(df)`. Bar fill = locked palette (group1=magenta, group2=cyan, additional groups cycle through platform categorical palette). Whiskers = mean ± 1 SD. Numeric `mean={x.xx}` and `SD={x.xx}` rendered as `fontSize: 0.85em` text below each bar (matches `uniprot-panel.ts:170`).
- **Empty state:** if no `proteomics.groups` tag OR all values are missing, render the missing-quantity message instead of an empty chart frame.
- **Async safety:** the panel already uses `ui.wait(async () => ...)`. Compute group means synchronously from the DataFrame (no network) — they are NOT part of the UniProt fetch.

### Group-Mean Correlation viewer (R4 / D-12)

- **Menu:** `Proteomics | Visualize | Group-Mean Correlation…` (sibling of Volcano Plot / Heatmap / PCA).
- **Factory:** `createGroupMeanCorrelation(df: DG.DataFrame, options?): DG.ScatterPlotViewer` in `src/viewers/group-mean-correlation.ts`. Pure factory (matches `createX` convention).
- **Derived columns added to `df` (in place, named):** `Numerator Mean`, `Denominator Mean` (use `SEMTYPE` constants — add to `proteomics-types.ts` + mirror in `detectors.js`).
- **Scatter config:** `x = 'Numerator Mean'`, `y = 'Denominator Mean'`, `color = direction` (the same column the volcano uses, after D-04 migration).
- **Title:** locked copywriting format (see Copywriting). Recompute on filter change.
- **Diagonal reference line:** add `y = x` formula line in `df.meta.formulaLines` colored `#888888` width 1 (matches volcano threshold lines).
- **Labels:** same `Display Name` top-15 logic as the volcano (consistent analyst muscle memory). Hover reveals `Source ID`.
- **Pearson/Spearman computation:** synchronous, over current filter. Use `@datagrok-libraries/statistics` if it exposes these; otherwise inline implementation in the factory file.
- **NOT in QC dashboard.** Hard separation per ROADMAP and D-12.

### Enrichment table: smart pathway filter banner (R5 / D-13 / D-14)

- **Dialog toggle:** one checkbox in the existing Enrichment dialog (`src/analysis/enrichment.ts` — wherever `showEnrichmentDialog` lives). Label: `"Apply smart pathway filter"`. Default checked. Tooltip per Copywriting.
- **Banner rendering:** when the filter dropped rows, render the banner copy as a single-line `ui.divText` strip docked ABOVE the enrichment grid in the same table view. Use platform info color (not destructive). Persist for the life of the table view; do not auto-dismiss.
- **Banner not rendered when:** filter was off, OR filter was on but dropped zero rows.
- **Set tag:** `df.setTag('proteomics.enrichment_smart_filtered', 'true')` so the banner can be re-derived without recomputing.

### Gene-label resolution display rules (R1 / D-07 / D-08 / D-10)

- **Display Name column** is the single canonical label everywhere a gene is shown: volcano labels, heatmap row headers, enrichment intersection lists, UniProt panel header, Group-Mean Correlation labels, grid cells.
- **Source ID column** is hidden from the default grid layout but accessible via column chooser. It is the hover/tooltip source for marker explanations.
- **Marker glyphs:**
  - `*` (asterisk) — grouped: one Ensembl/LOC ID resolved to a gene name shared by N>1 source IDs. Render as suffix: `Myh7*`.
  - `†` (dagger, U+2020) — predicted / reclassified: Ensembl returned a description rather than a gene name. Render as prefix on the readable description token: `Predicted-LOC123†`.
  - Both markers — apply both: `Myh7*†` (rare; defensive).
- **Disambiguation suffix (D-10):** when two rows resolve to the same gene name with different source IDs, append `(Source ID)`: `Myh7 (ENSRNOG00000001)` and `Myh7 (ENSRNOG00000099)`. This is OR-LOGIC with `*` — disambiguated names usually drop the `*` because they are no longer ambiguous after the suffix.
- **Tooltip:** `ui.tooltip.bind(...)` on the volcano hover already shows the label; ensure the tooltip text includes `"Original ID: {Source ID}"` for any row whose `Display Name !== Source ID`. Mechanism: planner decides between platform tooltip hooks and a custom format function.
- **No marker styling.** Plain text characters only. No color, no font-weight override, no superscript. They must round-trip through CSV export and grid copy unchanged.
