## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user file → browser parser | A user-selected multi-GB TSV (potentially hostile or malformed) is decoded and parsed entirely client-side in the browser. This is the primary untrusted-input boundary. |
| stream chunks → line buffer | Chunk boundaries split arbitrarily (mid-line, mid-UTF-8-codepoint); the carry-over buffer + `TextDecoderStream` reassemble across them. |
| header line → path routing | The first line's column names decide which code path runs (D-01 sniff via the exported `sniffIsPrecursor`). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-05 | Denial of service | unbounded memory from a hostile/huge report (the original V8-OOM bug) | mitigate | Single-pass aggregation: only state retained is the bounded carry-over line buffer + the agg `Map` keyed by (protein×condition×replicate) — bounded by #proteins×#samples, not input rows. Acceptance criterion explicitly forbids buffering raw lines/whole text (RESEARCH Pitfall 6). The bounded-aggregation invariant IS the mitigation. |
| T-12-06 | Denial of service | pathological line length / missing trailing newline / no newline at all | mitigate | A ~1 MB header-sniff sanity guard caps the pre-newline read; the main loop streams chunk-by-chunk (no per-line size assumption) and flushes the trailing partial line after `done` so a missing final newline does not drop or hang on data. |
| T-12-07 | Tampering | non-UTF-8 / chunk-boundary multibyte corruption | mitigate | `TextDecoderStream('utf-8')` is stateful and carries multi-byte sequences across chunk boundaries (WHATWG-spec correct); malformed bytes decode to U+FFFD rather than crashing. Malformed rows (field-count short / both casts null) are skipped + counted, not fatal (duckdb `ignore_errors` parity). |
| T-12-08 | Tampering / Spoofing | header column-name spoofing flips D-01 routing | accept | Worst case a PG-level file with a spurious `FG.Charge` column routes through the streaming path (still produces a correct PivotResult via the shared tail) or a precursor file lacking all three signature columns routes through `file.text()` (fails fast with the existing error + the new fallback hint).
