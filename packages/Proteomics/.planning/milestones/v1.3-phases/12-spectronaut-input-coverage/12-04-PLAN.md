<objective>
Close Phase 12 UAT Gap 1 (`status: failed`, in-scope): `parseSpectronautStream` in
`src/parsers/spectronaut-parser.ts` collapses four semantically distinct
`handleFields()` return-`false` outcomes into one `skipped` counter and reports it to
the user as "skipped N malformed line(s)". Two of those four outcomes — CON__/REV__
decoy/contaminant and numeric `EG.Qvalue > threshold` — are correct, intentional
duckdb-parity filtering, NOT malformed. On the 2.6 GB headline file this mislabels
millions of by-design-filtered precursor rows as "malformed", falsely signalling a
corrupt import on a provably-correct one.

Make `handleFields` return a discriminated outcome (`'kept' | 'malformed' |
'filtered'`), track two counters, and surface ONLY genuine-malformed as
"malformed/unparseable line(s)". The by-design-filtered category stays SILENT — the
text path (`pivotSpectronaut`, lines 47-80) applies the identical q-filter + CON__/REV__
drop with no message; matching that silence is the locked, path-consistent UX decision
(no open questions). Then add a regression test pinning the new behavior.

Purpose: the headline use case Phase 12 delivers (streaming a multi-GB correct
precursor report) must not show a false corruption signal. The aggregation output is
already provably correct (committed duckdb golden); this is a message/categorization
fix only.
Output: `src/parsers/spectronaut-parser.ts` (categorization + messages) and one new
test in `src/tests/spectronaut-parser.ts`.

Scope guard: this plan touches ONLY `src/parsers/spectronaut-parser.ts` and
`src/tests/spectronaut-parser.ts`. UAT gaps obs-A and obs-B are backlogged
(ROADMAP Phase 999.1 / 999.2), pre-existing, and out of Phase-12 scope —
`src/analysis/experiment-setup.ts` and `src/analysis/normalization.ts` are NOT touched.
