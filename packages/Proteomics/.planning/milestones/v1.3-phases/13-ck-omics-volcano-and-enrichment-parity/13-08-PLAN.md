<objective>
Close Gap 2 from 13-UAT.md: first-time Volcano Options → Color by location feels like a hang (no progress, ~80 strictly sequential UniProt round-trips, atomic-at-end cache, no short-circuit on repeat toggles). Add user-visible progress, bound concurrent UniProt chunks at 6, write the cache incrementally so interrupted sessions retain partial work, and short-circuit ensureLocationColumn when the accession set has not changed since the column was last populated. The spec was already functionally met in 13-UAT (locked 11-colour palette, Y on negLog10P, threshold at 1.301, direction populated) — this plan closes only the UX gap.

Purpose: Phase 13-04 implemented the network and cache contract; Phase 13-06 wired it to the dialog. The combination produces a multi-minute opaque wait on a real ~8k-accession Spectronaut Candidates file. The fix is mechanical — none of the gap_closure changes alter the locked 11-category classifier, the LOCATION_COLORS palette, the cache schema string, the userDataStorage key, or the D-03 reviewed-by-gene fallback semantics. Verbatim CK-omics parity is preserved.

Output: getSubcellularLocations accepts an optional `progress: ProgressCb` parameter, both fetch passes run via a bounded-concurrency helper (6 in-flight chunks), the cache is written periodically by a single wall-clock timer (Option A — no per-worker flush race), ensureLocationColumn stamps the accession-set hash on the column as a COLUMN tag and short-circuits on a hash match, volcanoOptions emits a pre-OK shell.info toast and routes pi.update through the threaded callback. Tests confirm the progress callback fires, the concurrency cap is respected, the periodic cache writes happen, and the column-tag short-circuit prevents re-walking.

Patterns established (new — for SUMMARY.md):
- Column-level tag convention. `proteomics.location_acc_hash` is set on the `Subcellular Location` DG.Column via `col.setTag/getTag`, NOT on the DataFrame via `df.setTag`. This is the first column-tag in the package (all prior `proteomics.*` workflow state is on the DataFrame). The convention: per-column cache-invalidation metadata belongs on the column; whole-pipeline workflow state belongs on the DataFrame.
- Timer-driven incremental flush. Concurrent workers should NEVER share mutex-style flush counters with `await` checkpoints — use a single setInterval timer in the orchestrator, stopped in `finally`, to flush wall-clock-periodically. See Action step 4.
