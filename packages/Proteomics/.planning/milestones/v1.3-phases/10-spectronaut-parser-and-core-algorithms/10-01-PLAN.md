Create `src/parsers/spectronaut-parser.ts` exporting `parseSpectronautText(text: string, qValueThreshold?: number): DG.DataFrame`. Follow the MaxQuant parser pattern but add a pivot step.

**Detection:** Check for required Spectronaut columns: R.FileName, R.Condition, R.Replicate, PG.ProteinGroups, PG.IBAQ.

**Pivot algorithm:**
1. Parse TSV via `DG.DataFrame.fromCsv(text, {delimiter: '\t'})`
2. Iterate rows: build `Map<protein, Map<sampleKey, ibaqValue>>` where sampleKey = `${R.Condition}_${R.Replicate}`
3. Q-value filter: skip rows where numeric EG.Qvalue > qValueThreshold (default 0.01); non-numeric values treated as passing
4. CON__/REV__ filter on PG.ProteinGroups prefix
5. First-encountered value wins for each protein+sample (PG.IBAQ is constant per protein+sample -- deduplication not aggregation)
6. Collect PG.Organisms per protein, R.FileName per sampleKey

**Build wide DataFrame:**
1. `DG.Column.fromStrings('PG.ProteinGroups', proteins)` with `semType = SEMTYPE.PROTEIN_ID`
2. For each sampleKey: `DG.Column.fromFloat32Array(sampleKey, values)` with `semType = SEMTYPE.INTENSITY`
3. Set `spectronaut.fileName` tag on each intensity column from collected R.FileName mapping
4. Add PG.Organisms column if present
5. `DG.DataFrame.fromColumns(cols)`

**Post-pivot processing (mirror MaxQuant pattern):**
1. `addPrimaryColumnIfNeeded()` for semicolon-delimited protein groups
2. `detectLog2Status()` on intensity columns
3. If raw: `log2TransformColumns()` to create log2 columns, keeping originals
4. If log2: `copyAsLog2Columns()` -- per CONTEXT.md decision for pre-normalized data
5. Tag `proteomics.preNormalized = 'true'` if detected as log2
6. Tag `proteomics.source = 'spectronaut'`

**Auto-group population:**
1. Extract unique R.Condition values from sampleKeys (split on last underscore)
2. If exactly 2 conditions: call `setGroups(df, {group1: {name, columns: log2ColNames}, group2: {name, columns: log2ColNames}})`
3. If >2 conditions: skip auto-assignment (user picks in annotation dialog)

**Tests:** Create `src/tests/spectronaut-parser.ts` with `category('Spectronaut', ...)`. Build test helper `makeLongFormatTsv()` that generates synthetic Spectronaut data with known protein counts, conditions, replicates, q-values, and contaminant rows. Register in `package-test.ts` via `import './tests/spectronaut-parser'`.

Use `@datagrok-libraries/test/src/test` for `category`, `test`, `expect` imports (same pattern as existing parsers.ts tests).

```typescript
@grok.decorators.func({'top-menu': 'Proteomics | Import | Spectronaut...'})
static async importSpectronaut(): Promise<void> {
  DG.Utils.openFile({
    accept: '.tsv,.txt,.csv',
    open: async (file: File) => {
      const text = await file.text();
      const df = parseSpectronautText(text);
      df.name = file.name.replace(/\.[^.]+$/, '');
      grok.shell.addTableView(df);
      grok.shell.info(`Imported ${df.rowCount} protein groups from Spectronaut`);
    },
  });
}
```

This mirrors the existing `importMaxQuant` handler pattern exactly.
