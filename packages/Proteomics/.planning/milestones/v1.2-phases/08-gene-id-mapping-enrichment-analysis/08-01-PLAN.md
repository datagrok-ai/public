Create `packages/Proteomics/src/analysis/enrichment.ts` with all enrichment functionality in a single module. Follow the research patterns from 08-RESEARCH.md exactly. The module must contain:

**Constants and Types:**
- `GPROFILER_BASE = 'https://biit.cs.ut.ee/gprofiler'`
- `ORGANISM_LIST` array of 9 organisms with `{display, code}` fields (human, mouse, rat, yeast, E. coli, zebrafish, Drosophila, Arabidopsis, C. elegans) -- use the exact codes from 08-RESEARCH.md
- `ConvertResult` interface (incoming, converted, name, description, namespaces, n_incoming, n_converted)
- `GostResult` interface (native, name, source, p_value, significant, term_size, query_size, intersection_size, effective_domain_size, precision, recall, intersections)

**API Client Functions:**
- `fetchWithTimeout(url, options, timeoutMs=30000)` -- wraps fetch with AbortController timeout per research pattern. On AbortError, throw descriptive message about g:Profiler availability. On non-ok response, throw with status code.
- `gConvert(accessions: string[], organism: string = 'hsapiens'): Promise<ConvertResult[]>` -- POST to `/api/convert/convert/` with organism, query, target:'ENSG', numeric_ns:'', output:'json'. Returns `data.result ?? []`.
- `gGOSt(queryGenes: string[], backgroundGenes: string[], organism: string, sources: string[], threshold: number = 0.05): Promise<GostResult[]>` -- POST to `/api/gost/profile/` with all params including `domain_scope: 'custom'`, `background: backgroundGenes`, `all_results: true`, `no_evidences: false`, `significance_threshold_method: 'fdr'`. Returns `data.result ?? []`. NOTE: The g:GOSt response structure nests results differently -- check research carefully. The actual results array is at `data.result[0].result` or similar. Handle the case where the API returns `data.result` as an array of query results. Extract the flat list of term results from the nested structure.

**Result Builder:**
- `buildEnrichmentDf(results: GostResult[], queryGenes: string[]): DG.DataFrame` -- Creates DataFrame with locked schema columns: Source (string), Term ID (string), Term Name (string), P-value (float), FDR (float), Gene Count (int), Gene Ratio (float), Intersection (string). For P-value and FDR: g:GOSt with fdr method returns adjusted p-values in `p_value` field -- set both P-value and FDR to `r.p_value`. Gene Count = `r.intersection_size`. Gene Ratio = `r.precision`. Derive Intersection member genes from positional intersections array matched against queryGenes. Add boolean 'Significant' column (FDR < 0.05). Apply color coding on FDR column. Set `df.setTag('proteomics.enrichment', 'true')`. Set `df.name = 'Enrichment Results'`.

**Significant Protein Counter:**
- `countSignificantProteins(df: DG.DataFrame, fcThreshold: number, pThreshold: number, log2fcCol: DG.Column, pValCol: DG.Column): {significant: number, total: number}` -- Counts proteins meeting |log2FC| >= fcThreshold AND adj.p <= pThreshold. Used by dialog for live count.

**Pipeline Orchestrator:**
- `runEnrichmentPipeline(df: DG.DataFrame, fcThreshold: number, pThreshold: number, organismCode: string, sources: string[]): Promise<DG.DataFrame>` -- Full pipeline per research pattern:
  1. Find proteomics columns via `findProteomicsColumns(df)`
  2. Check if geneName column exists with SEMTYPE.GENE_SYMBOL -- if yes, use directly
  3. If no gene names, extract clean accessions via `parseAccession()` from protein ID column, call `gConvert()`, build accession-to-gene map (first match per accession), add 'Gene Symbol (mapped)' column with semType GENE_SYMBOL to source df
  4. Build significant gene set (meeting thresholds) and background gene set (all mapped genes) -- deduplicate both
  5. Call `gGOSt()` with query=significant, background=all, organism, sources
  6. Build result DataFrame via `buildEnrichmentDf()`
  7. Return the enrichment DataFrame
  - Handle one-to-many mapping by taking first match per accession (per research guidance)
  - Return mapping statistics as `{mapped, total, unmapped}` alongside the result (either as a second return value via object or by reporting in progress indicator)

**Dialog:**
- `showEnrichmentDialog(df: DG.DataFrame): void` -- Per locked decisions:
  1. Guard: check `df.getTag('proteomics.de_complete') !== 'true'` -- warn and return if DE not run
  2. Find log2fc and pValue columns via `findProteomicsColumns(df)`
  3. Create inputs: `ui.input.float` for |log2FC| threshold (default 1.0), `ui.input.float` for adj.p-value threshold (default 0.05)
  4. Create `ui.input.choice` for organism (items from ORGANISM_LIST display names, default 'Homo sapiens (Human)')
  5. Create `ui.input.bool` for each source: GO:BP, GO:MF, GO:CC, KEGG, Reactome -- all default true
  6. Create `countDiv = ui.divText('')` for live count display
  7. Wire `updateCount` function on fcInput.onChanged and pInput.onChanged -- calls `countSignificantProteins()` and updates countDiv text to format: "42 of 1,200 proteins are significant with these thresholds"
  8. Call `updateCount()` once immediately
  9. Build dialog via `ui.dialog('Enrichment Analysis')` with all inputs
  10. onOK: create `DG.TaskBarProgressIndicator.create('Running enrichment analysis...')`, resolve organism code from display name, build sources array from checkbox states (map to 'GO:BP', 'GO:MF', 'GO:CC', 'KEGG', 'REAC'), call `runEnrichmentPipeline()`, show mapping stats in info toast, call `grok.shell.addTableView(resultDf)`, close progress indicator in finally block
  11. Handle errors: catch API failures, show `grok.shell.error()` with descriptive message

**Imports:**
- `import * as grok from 'datagrok-api/grok'`
- `import * as ui from 'datagrok-api/ui'`
- `import * as DG from 'datagrok-api/dg'`
- `import {parseAccession} from '../panels/uniprot-panel'`
- `import {findProteomicsColumns} from '../utils/column-detection'`
- `import {SEMTYPE} from '../utils/proteomics-types'`

Export `showEnrichmentDialog` as the public API for menu wiring. Also export `buildEnrichmentDf`, `countSignificantProteins`, `ORGANISM_LIST`, `gConvert`, and `gGOSt` for testing.
