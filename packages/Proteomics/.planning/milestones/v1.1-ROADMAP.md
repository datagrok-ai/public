## Phase Details

### Phase 6: Generic Matrix Parser
**Goal**: Scientists can import any CSV/TSV proteomics matrix into the full analysis pipeline, not just MaxQuant output
**Depends on**: Phase 5 (existing import infrastructure and semantic types)
**Requirements**: IMPORT-01, IMPORT-02, IMPORT-03, IMPORT-04, IMPORT-05, IMPORT-06
**Success Criteria** (what must be TRUE):
  1. User can open a CSV or TSV file and have it recognized as a proteomics matrix via file handler
  2. User sees a column mapping dialog with auto-suggested protein ID and intensity columns, plus a data preview of the first rows
  3. User can confirm or override column mapping and optionally enable log2 transform before import completes
  4. After import, the resulting DataFrame has the same semantic types as MaxQuant import, and the full downstream pipeline (annotation, normalization, imputation, DE) works without modification
**Plans:** 2 plans

Plans:
- [ ] 06-01-PLAN.md -- Extract shared utilities, refactor MaxQuant parser, create generic parser test scaffold
- [ ] 06-02-PLAN.md -- Build generic import dialog with column mapping, add menu entry, backport MaxQuant source tags

### Phase 7: QC Dashboard
**Goal**: Scientists can assess data quality through a unified dashboard of linked QC viewers before and after normalization
**Depends on**: Phase 6 (generic parser broadens data that needs QC; also needs group annotations for MA/CV plots)
**Requirements**: QC-01, QC-02, QC-03, QC-04, QC-05, QC-06
**Success Criteria** (what must be TRUE):
  1. User can open a QC dashboard from the ribbon/menu that displays all QC viewers in a single linked layout
  2. User can see an MA plot showing intensity-dependent bias between experimental conditions
  3. User can see missing value patterns, sample-to-sample correlations, and per-sample intensity distributions as linked viewers
  4. User can see a CV plot showing measurement variability per protein within replicates
  5. Selecting proteins or samples in one QC viewer highlights them across all other viewers in the dashboard
**Plans**: 3 plans

Plans:
- [ ] 07-01-PLAN.md -- QC computation utilities and dashboard orchestration with viewer layout
- [ ] 07-02-PLAN.md -- Menu entry wiring, unit tests, and package build verification
- [ ] 07-03-PLAN.md -- Gap closure: fix auxiliary DataFrame viewer creation (heatmap, bar chart, box plot)

### Phase 8: Gene ID Mapping & Enrichment Analysis
**Goal**: Scientists can map protein IDs to gene symbols and run GO/KEGG/Reactome overrepresentation analysis on their DE results, with results displayed in an interactive table
**Depends on**: Phase 5 (DE pipeline produces the protein list and fold-change/p-value columns that enrichment operates on)
**Requirements**: MAP-01, MAP-02, ENRICH-01, ENRICH-02, ENRICH-03, VIZ-01
**Success Criteria** (what must be TRUE):
  1. User can map UniProt accessions to gene symbols with mapping statistics reported (success rate, unmapped proteins listed), and select organism (defaulting to human)
  2. User can run GO overrepresentation analysis (BP, MF, CC) on significant DE proteins, using all quantified proteins as the background set (not whole genome)
  3. User can run KEGG and Reactome pathway enrichment on significant DE proteins with FDR-corrected p-values
  4. User can view enrichment results in an interactive table with term ID, description, p-value, FDR, gene count, and member genes -- sortable and filterable
  5. Enrichment results are stored in a separate DataFrame with a standard schema, enabling downstream visualization
**Plans:** 2/2 plans complete

Plans:
- [ ] 08-01-PLAN.md -- g:Profiler API client, enrichment dialog, pipeline orchestration, result builder, and unit tests
- [ ] 08-02-PLAN.md -- Menu entry wiring, package.g.ts regeneration, and full build verification

### Phase 9: Enrichment Visualization & Volcano Integration
**Goal**: Scientists can visually explore enrichment results through charts and link enrichment terms back to proteins on the volcano plot
**Depends on**: Phase 8 (enrichment DataFrame must exist with standard schema)
**Requirements**: VIZ-02, VIZ-03, ENRICH-04
**Success Criteria** (what must be TRUE):
  1. User can view a dot plot of top enriched terms showing gene ratio, gene count, and adjusted p-value
  2. User can view a bar chart of top enriched terms ranked by significance or gene count
  3. User can select a GO or pathway term in the enrichment results and see its member proteins highlighted on the volcano plot
  4. Visualizations show a compact top-N view by default to avoid overwhelming the user with redundant GO terms
**Plans:** 2/2 plans complete

Plans:
- [ ] 09-01-PLAN.md -- Enrichment visualization module with dot plot, bar chart, cross-DF wiring, and unit tests
- [ ] 09-02-PLAN.md -- Wire into enrichment dialog, add menu entry, rebuild package
