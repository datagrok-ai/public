### Phase 999.4: Spectronaut streaming↔text malformed-counter parity (HOTFIX)

**Goal:** Phase 12-04 introduced the `LineOutcome` discriminated drop classification but deliberately left empty-`PG.ProteinGroups` and both-numeric-casts-null rows in the user-surfaced `'malformed'` category, even though the text path (`pivotSpectronaut`) drops both silently. Spectronaut routinely emits unassigned-precursor rows, so the streaming path inflated the "skipped N malformed line(s)" message on perfectly normal output. Fix: reclassify both as `'filtered'` (silent); the malformed counter now fires only for structurally truncated rows, matching the text path exactly. Aggregation output is byte-identical (duckdb-golden equivalence test stays green). Severity: MEDIUM (user-message noise / false corruption signal). Surfaced by Phase 12 code review (12-REVIEW.md WR-01/WR-03).

### Phase 999.3: Publish read-only proteomics analysis for biologist review (BACKLOG)

**Goal:** [Captured for future planning] Paired user-stories — (a) "As a proteomics expert, I should be able to publish a read-only analysis that biologists will interrogate"; (b) "As a biologist, I should be able to interrogate a proteomics experiment by examining the volcano plot and enrichments without concern that I will change its content." A proteomics author who has completed the full pipeline (parse → annotate → normalize → impute → DE → enrichment → volcano + heatmap + correlation) currently has no first-class way to hand the analysis to a biologist consumer as a frozen, read-only artifact: the only path is to share the live DataFrame, which exposes raw intensities and lets the consumer accidentally re-run normalization, change DE parameters, or mutate columns. Likely solution shape (per the detailed scoping in `.planning/todos/pending/2026-05-11-share-analysis-read-only-with-biologics-team-filed-by-target.md`): a "Proteomics → Share Analysis for Review…" menu entry that builds a trimmed shareable project (protein ID, gene symbol, log2FC, p, adj.p, sig flag, optionally enrichment terms — no raw intensities), files it under a target-keyed Datagrok Space, and applies read-only permissions to the biologist reviewer group. Severity: STRATEGIC (cross-team-workflow enabler; likely the spine of a v1.5 "Cross-team review" milestone). Surfaced by Cytokinetics meeting 2026-05-04; re-confirmed 2026-06-03 with explicit producer + consumer user-stories.
