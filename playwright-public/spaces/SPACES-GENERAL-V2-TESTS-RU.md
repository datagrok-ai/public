# spaces-general-v2.test.ts — Детальна документація тестів

Наскрізні (end-to-end) Playwright-тести для функціональності **Spaces** в Datagrok. Набір покриває повний життєвий цикл просторів: створення, ієрархію, валідацію, перейменування, видалення, обране, шарінг, пошук, drag-and-drop (копіювання/переміщення), взаємодію з контент-представленням, операції з сутностями та граничні випадки.

**Цільовий URL:** `DATAGROK_URL` з `.env` (напр. `https://dev.datagrok.ai`)  
**Браузер:** Chromium (headless)  
**Воркери:** 1 (послідовне виконання)  
**Таймаут за замовч.:** 60с на тест (деякі перевизначають на 120–180с)  
**Авторизація:** Попередня автентифікація через `globalSetup` → зберігається в `e2e/.auth.json`  
**Користувач для шарінгу:** `DATAGROK_SHARING_LOGIN` / `DATAGROK_SHARING_PASSWORD` з `.env`

---

## Допоміжні функції (хелпери)

### `openSpacesView(page)`
Переходить на головну сторінку Datagrok і чекає поки вузол **Spaces** стане видимим у дереві навігації (до 20с).

### `refreshSpacesTree(page)`
Натискає іконку sync (`i.grok-icon[name="icon-sync"]`) для оновлення дерева. Використовується після мутацій для перевірки що refresh не відкочує стан UI.

### `apiDeleteSpace(page, name)`
Видаляє простір за ім'ям через `grok.dapi.spaces`. Використовується в блоках `finally` для очищення.

### `rightClickSpacesTreeNode(page)`
Відправляє `contextmenu` на кореневий вузол "Spaces". Використовує `dispatchEvent` щоб обійти тултіпи.

### `rightClickSpace(page, name)` / `clickMenuItem(page, label)` / `fillNameAndOk(page, name)`
Правий клік на простір / клік по пункту меню / заповнення діалогу.

### `uiCreateRootSpace(page, name)`
Правий клік "Spaces" → "Create Space..." → ім'я → перевірка в дереві.

### `openSpaceContent(page, spaceName)` / `openSpaceViaTree(page, spaceName)`
Відкриття контент-представлення простору (з перезавантаженням сторінки / без).

### `dragFileToSpaceNode(page, fileName, spaceName)`
Переходить на DemoFiles, перевіряє наявність файлу (якщо немає — кидає помилку з переліком доступних), перетягує на вузол простору.

### `addFileToSpaceViaCopy(page, fileName, spaceName)`
Обгортка `dragFileToSpaceNode` + Copy. **Важливо:** всі операції з DemoFiles використовують **Copy** (не Link) щоб не видалити оригінали.

### `rightClickItemInSpaceView(page, name)`
Правий клік на елемент у контент-представленні простору.

## Ізоляція даних

- Всі імена просторів використовують префікс `PW-Gen-*` з номером тесту
- Всі тести очищують через `try/finally` + `apiDeleteSpace`
- Тести 9, 10, 16 також попередньо видаляють простори на початку
- Всі операції з DemoFiles використовують **Copy** (не Link) для захисту оригіналів

## Ключові рішення

- **Паттерн `refreshSpacesTree`:** Перевірка результату БЕЗ refresh, потім refresh і перевірка що результат не змінився
- **DnD retry:** До 3 спроб для DnD операцій що можуть не спрацювати через тултіпи
- **Перевірка файлів DemoFiles:** `dragFileToSpaceNode` кидає описову помилку якщо файл не знайдено
- **Без Link з DemoFiles:** Всі `addFileToSpaceViaLink` замінені на `addFileToSpaceViaCopy`
