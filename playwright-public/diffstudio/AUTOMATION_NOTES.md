# Как 13 manual-кейсов превратились в автотесты

Краткая шпаргалка по второму проходу автоматизации Diff Studio Playwright-тестов.
Каждый раздел: **что делает QA глазами** → **как это автоматизировано**.

---

## 1. Глобальный мониторинг ошибок (M-8)
**Глазами:** открыть DevTools и смотреть в console + Network + балуны на каждом шаге.

**Кодом:** `page.on('console')` + `page.on('pageerror')` + `page.on('response')` ловят всё
автоматически. Список "шумных" пакетов (Jiraconnect, Plates, WASM-aborted) в фильтре — чтобы
не падать на чужих багах.

---

## 2. Live chart redraw (M-1.1 – M-1.4)
**Глазами:** двинул слайдер → посмотрел, перерисовался ли график.

**Кодом:** канвас умеет отдать свои пиксели — `canvas.toDataURL()`. Хэшнул его до изменения,
поменял инпут, хэшнул после. Хэши разные = пиксели разные = график перерисовался. Просто и
надёжно.

---

## 3. 12 разных цветов в Facet (M-2)
**Глазами:** посмотреть, что соседние подграфики не одинакового цвета.

**Кодом:** прошёл по всем пикселям канваса через `getImageData`. Выкинул белые/чёрные/серые.
Каждый цвет огрубил до 4-битов на канал (чтобы не считать антиалиасинг новым цветом). Посчитал
уникальные ведра. Получилось ≥10 = много разных цветов.

---

## 4. Тултип говорит что-то осмысленное (M-3.1 / M-3.2)
**Глазами:** прочитать тултип, понять что это про правильную переменную.

**Кодом:** в IVP-файле тултип написан прямо в `[квадратных скобках]`. Прочитал файл через API
(`grok.dapi.files.readAsText`), регексом вытащил все `[tooltips]`, потом сравнил с тем, что
показал UI на hover. Совпадает = тест зелёный. Если файл недоступен — hardcoded fallback.

---

## 5. Target Block + RMSE descending (M-4)
**Глазами:** прокрутить вниз формы, потыкать switchers, запустить, посмотреть что RMSE-кривая
падает.

**Кодом:** все switcher'ы — `.sa-switch-input` элементы в DOM. Просто посчитал их и тогнул
лишний (после уже включённых). После Run прочитал результирующий dataframe через
`grok.shell.t`, нашёл колонку с "RMSE" в имени, проверил что значения падают (95%+ соседних
пар идут вниз).

---

## 6. 4 SA viewer'а валидные (M-5)
**Глазами:** посмотреть каждый из 4 viewer'ов, нет ли NaN, осмысленны ли оси.

**Кодом:** `grok.shell.v.viewers` отдаёт массив всех viewer'ов с их dataFrame. Прошёлся по
каждому, проверил что в numerical колонках нет NaN/Inf, имя viewer'а не "untitled".

---

## 7. Перетащить слайдер реально мышью (M-6)
**Глазами:** зажать кнопку мыши на ползунке и протащить.

**Кодом:** `page.mouse.move/down/move/up`. Взял bounding-box слайдера, двинул мышь в левый
край → нажал → 10 промежуточных move'ов → отпустил. Получил настоящий drag, а не `.fill()`.

---

## 8. Refresh реально перезагружает (M-7)
**Глазами:** удалить файл во второй вкладке, в первой нажать Refresh, посмотреть что карточка
исчезла.

**Кодом:** в той же вкладке через `grok.dapi.scripts.delete` удалил user-копию PK-PD. Посчитал
карточки ДО. Кликнул Refresh. Посчитал ПОСЛЕ. Разница = ровно то что удалил.

---

# Принцип

Везде один и тот же приём: **найти что-то измеримое, что меняется при том событии, которое
хочется проверить**.

| Что хочется проверить | Что измеримо |
|---|---|
| Цвета                 | пиксели                       |
| Перерисовка           | хэш канваса                   |
| Содержимое тултипа    | исходник IVP                  |
| Данные viewer'а       | JS API даёт DataFrame напрямую |
| Drag                  | синтетический mouse-event     |
| Refresh               | дельта счётчика               |

Где **ничего из этого не работает** (3 кейса с Compute2 RFV — Vue прячет всё), — там реально
невозможно и остаётся ручная проверка. Это уже не "лень", а архитектурное ограничение
Vue-инкапсуляции.
